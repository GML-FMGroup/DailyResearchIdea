# Compositional Latent Memory Agents for Long-Horizon Reasoning in Open-Ended Environments

## Motivation

The AgentOdyssey benchmark reveals that current agents struggle with long-horizon tasks and retention of episodic memory, primarily due to the lack of a compositional latent memory that can compress and retrieve intermediate reasoning steps. Even state-of-the-art LLM-based agents show weak scaling with model size on these tasks, indicating a structural bottleneck in memory architecture rather than representational capacity.

## Key Insight

By introducing a generative latent memory module that learns to compose sequences of observations into hierarchical latent tokens, the agent can maintain a compact, reusable representation of past events that supports efficient retrieval for future decisions, enabling hierarchical reasoning over hundreds of time steps.

## Method

**CLMA: Compositional Latent Memory Agent**

(A) **What it is** – CLMA augments a base LLM agent with a generative latent memory module that uses a trigger/weaver mechanism (inspired by MemGen) to compress observations into latent tokens and retrieve them compositionally. Input: current observation o_t, previous hidden state h_{t-1}, memory bank M. Output: action a_t, updated memory bank M.

(B) **How it works** (pseudocode):
```python
# CLMA step
# Encoder: 2-layer MLP (hidden=256, GeLU) f_enc(o_t, h_{t-1}) -> z_t (latent at time t)
# Structured key-value memory: each memory token has key split into m=4 subspaces, each subspace has own bilinear attention with learned W_i
for each candidate i in M:
    # Multiplicative gating over subspaces
    for subspace s in 1..4:
        w_i_s = softmax(z_t^T W_s q_i_s)
    g_i = prod_s w_i_s   # element-wise product
    w_i = softmax(g_i)   # final attention weight
# Retrieval: context vector r_t = sum_i w_i * v_i (values)
# Trigger: generate new latent token if importance > threshold
importance = 1 - max(w)        # novelty: low attention to existing token
if importance > θ_imp (0.7):
    c_t = 2-layer MLP(hidden=256, GeLU)([z_t, r_t])      # new token (key= c_t, value=c_t)
    append c_t to M
# Decoder: action = LLM_policy (Phi-3 mini) ([o_t, r_t, h_{t-1}])
# Every 50 steps, compress M via k-means (K=1000) to merge similar tokens
```
(C) **Why this design** – We chose a generative latent memory over explicit episodic buffers because latent compression enables the agent to abstract away irrelevant details, improving retrieval relevance over long horizons — the cost is that the latent representation may lose fine-grained information, which we mitigate by allowing retrieval of multiple tokens with attention weights. We chose bilinear attention over cosine similarity because bilinear captures interactions between query and key features, at the cost of additional learned parameters, but this provides compositional retrieval (matching combinations of features) rather than simple nearest-neighbor. We chose to generate new latent tokens only when an importance score exceeds a threshold, rather than at every step, to control memory growth and avoid redundancy; the trade-off is that infrequent events may be missed if thresholds are too high. We chose to periodically compress the memory bank via clustering to maintain bounded size while preserving diverse experiences; the trade-off is potential loss of distinct memory traces, but this is acceptable because compression merges similar experiences into a prototypical latent.

(D) **Why it measures what we claim** – The retrieval compatibility score (structured bilinear attention with multiplicative gating) measures compositional relevance because it computes the match between the current latent query and memory keys as a weighted sum over subspace interactions, under the assumption that the latent space is structured such that feature interactions are linearly separable (A). This assumption fails when the latent space is entangled (F), in which case retrieval becomes nearest-neighbor with extra parameters. The importance score for memory addition measures the novelty of the new experience relative to existing memory, because it is computed as the negative maximum attention weight to existing tokens (if high overlap, low importance), under the assumption that novel experiences are most valuable for future generalization; this assumption fails when rare but critical events are also novel, then the score correctly identifies them, but if a rare event is very similar to common events (e.g., same action different context), it might be incorrectly pruned. The latent token itself measures compressed episode-level knowledge because it is generated by an MLP that combines the current observation and retrieved context, under the assumption that the MLP learns a compact representation of the state-action history; this assumption fails when the MLP capacity is insufficient to capture long-range dependencies, in which case the latent reflects only recent history.

**Load-bearing assumption:** This design relies on the assumption that the latent space is structured such that feature interactions are linearly separable, enabling the structured bilinear attention with multiplicative gating to capture compositional relevance. If the latent space is entangled, retrieval collapses to nearest-neighbor with extra parameters. To verify, we include a calibration experiment on synthetic data (see experiment).

**Implementation details:** Training on 4 A100 GPUs for 72 hours. LLM policy: Phi-3 mini (2.7B parameters) frozen; only memory module parameters (encoder, retrieval, trigger, compression) are trained.

## Contribution

(1) A novel agent architecture, Compositional Latent Memory Agent (CLMA), that integrates a generative latent memory module with compositional retrieval into an LLM-based policy for open-ended long-horizon environments. (2) An empirical finding that CLMA significantly improves episodic memory retention and task success rates over standard memory-augmented agents on the AgentOdyssey benchmark, and that its performance scales with memory capacity rather than model size. (3) A set of diagnostic metrics for analyzing hierarchical reasoning in text-game agents, including latent token coverage and retrieval precision.

## Experiment

### Evaluation Setup

| Role | Choice | Rationale (≤12 words) |
| --- | --- | --- |
| Dataset | AgentOdyssey text games | Open-ended long-horizon procedural generation |
| Primary metric | Game progress score | Measures task completion over many steps |
| Baseline: No memory | Static policy | No learning or adaptation |
| Baseline: Episodic buffer | Store all past observations | Baseline for explicit memory retrieval |
| Baseline: Parameter update | Online fine-tuning of LLM | Baseline for weight-based learning |
| Baseline: RAG | Retrieval-Augmented Generation | State-of-the-art memory-augmented agent |
| Ablation of ours | CLMA without k-means compression | Tests necessity of bounded memory |
| Ablation: Random attention | Replace structured attention with fixed random weights | Tests if learned attention drives performance |
| Ablation: Concatenation baseline | Replace attention with query-key concat + MLP | Tests if compositionality requires explicit interaction |
| Synthetic sanity-check | Semi-structured environment with known compositional vs. non-compositional recall tasks | Direct measure of compositional retrieval capability |

### Why this setup validates the claim

The combination of AgentOdyssey's procedurally generated long-horizon tasks and the chosen baselines tests each core hypothesis of CLMA. The no-memory baseline isolates the need for any memory; the episodic buffer tests explicit storage versus latent compression; parameter update tests whether weight adaptation is sufficient; the RAG baseline calibrates against a modern memory-augmented agent; the ablations isolate the effect of compositional retrieval and bounded memory. The synthetic sanity-check provides a controlled test of compositional vs. non-compositional retrieval, reporting correlation between attention weights and ground-truth composition. The primary metric, game progress, directly reflects the agent's ability to accumulate and use knowledge over time, which is the central claim. If CLMA outperforms all baselines and ablations, and the synthetic check shows high correlation, the claim that compositional latent memory with compression is effective is supported. Conversely, if a baseline matches CLMA, the unique advantage of our mechanism is falsified.

### Expected outcome and causal chain

**vs. No memory** — On a case where solving a new puzzle requires recalling a tool seen 100 steps earlier, the static policy produces random actions because it has no mechanism to store or retrieve past info. Our method instead retrieves the latent token encoding that tool via structured attention, so we expect a significant gap on tasks requiring long-term recall, but parity on short-horizon tasks.

**vs. Episodic buffer** — On a case where the agent must combine features (e.g., red key with blue door) from different past experiences, the episodic buffer performs exact retrieval, often missing relevant composite patterns. Our method computes compositional compatibility via subspace gating, enabling flexible combination, so we expect a larger gap on compositional tasks than on simple recall.

**vs. Parameter update** — On a case where learning a new skill would overwrite previously learned task (catastrophic forgetting), online fine-tuning degrades performance on old tasks. Our method stores latent tokens separately and only updates memory via addition, preserving past knowledge, so we expect higher retention on earlier tasks after learning new ones.

**vs. RAG** — On tasks requiring long-horizon reasoning, RAG with dense passage retrieval may retrieve irrelevant chunks due to lack of compression. Our method's latent tokens abstract away noise, so we expect an advantage on tasks with many distractors. On simple look-up tasks, RAG may match or exceed CLMA.

**vs. Ablation (no compression)** — On a long trajectory with many redundant observations, the unbounded memory grows linearly, slowing retrieval and eventually hurting performance. Our method compresses via k-means, bounding memory size and maintaining speed, so we expect a widening gap as episode length increases.

**vs. Random attention & Concatenation baseline** — On synthetic compositional tasks, random attention fails because weights are not tuned; concatenation baseline lacks explicit feature interaction, so performs worse on compositional but may be competitive on non-compositional tasks. We expect CLMA to excel on compositional tasks, validating the need for structured attention.

**Synthetic sanity-check** — We construct a grid world where each observation has two attributes (color and shape). Compositional recall requires retrieving a combination (e.g., red circle), while non-compositional recall requires single attribute (e.g., any red object). We measure the correlation between attention weights and ground-truth composition (e.g., subspace gating weights should be high only when both attributes match). Expected: high Pearson correlation (>0.7) for compositional, low for non-compositional. If correlation is low, the attention mechanism does not capture compositionality.

### What would falsify this idea

If our method's gain over the episodic buffer is uniform across both simple recall and compositional tasks, rather than concentrated on compositional ones, the structured attention mechanism is not providing the claimed advantage. Alternatively, if the no-compression ablation matches or exceeds the full method on long episodes, the compression is harming rather than helping. If the synthetic sanity-check shows low correlation between attention weights and ground-truth composition, the compositional retrieval claim is falsified.
