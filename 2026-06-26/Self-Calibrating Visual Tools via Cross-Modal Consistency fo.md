# Self-Calibrating Visual Tools via Cross-Modal Consistency for Robust Video Understanding

## Motivation

Current confidence-aware tool orchestration methods like Robust-TO assume that tool confidence scores are accurate, but in practice tools are systematically miscalibrated due to domain shifts and training-data biases. This miscalibration leads to suboptimal tool selection and evidence weighting, degrading performance under perturbations. The root cause is the lack of a label-free signal to detect and correct miscalibration at test time.

## Key Insight

Cross-modal consistency between visual tool outputs and textual descriptions provides a natural and unbiased proxy for calibration error because systematic miscalibration induces systematic inconsistency across modalities that share semantic content.

## Method

### (A) What it is
**Cross-Modal Consistency Calibration (CM³)** is a self-supervised calibration module that takes tool predictions (labels and confidence scores) and a set of textual descriptions generated by diverse vision-language models (VLMs) as input, and outputs recalibrated confidence scores for each tool, all without requiring labeled calibration data.

### (B) How it works
```python
# Input: list of tool outputs T = [t_i], each with (label, confidence) and video frames V
# Generate M=3 textual descriptions using frozen VLMs: Video-LLaVA, BLIP-2, LLaVA-NeXT-Video (different checkpoints)
D_set = []
for vlm in [VideoLLaVA, BLIP2, LLaVANextVideo]:
    D_set.append(generate_text(V, query=None, model=vlm))

# For each tool output t_i:
# 1. Encode tool label and each description into embeddings using BERT-base (768-dim)
e_label = encode(t_i.label)  # CLS token
# 2. Compute cosine similarity per description, then average and variance
c_i_m = [cosine_sim(e_label, encode(D)) for D in D_set]
c_i = mean(c_i_m)  # in [0,1], manually scaled from [-1,1]
v_i = var(c_i_m)  # variance as uncertainty indicator
# 3. Calibrate confidence: adjust original confidence s_i to s_i' using a logistic function with parameters learned per tool.
#    For each tool, maintain a buffer of N=1000 recent (c_i, correctness_approx) pairs.
#    correctness_approx = 1 if t_i.label has ROUGE-L > 0.5 with any description, else 0.
#    Every 100 instances, fit a logistic regression: correctness_approx ~ sigmoid(w0 + w1*c_i + w2*(1-v_i))
#    using scipy's minimize with L-BFGS, max 100 iterations.
#    At inference, compute:
#        s_i' = sigmoid(w0 + w1*c_i + w2*(1-v_i))
#        if v_i > 0.1: s_i' = s_i' * 0.5  # downweight due to potential hallucination
#    Output recalibrated confidence s_i' for tool t_i.
```

### (C) Why this design
We chose a logistic recalibration function over more complex neural networks because interpretability and low sample efficiency are critical—our calibration signal (consistency) is itself noisy, and a high-capacity model would overfit to spurious correlations; the cost is that the linearity assumption may not capture non-linear miscalibration patterns, but in practice we find it sufficient (see experiments). We selected a fixed text encoder (BERT) over fine-tuning it jointly because any adaptation would risk encoding tool-specific biases into the consistency measure; the drawback is that BERT embeddings may not perfectly align with tool label semantics, but we mitigate this by using a large pre-trained model. We use M=3 diverse VLMs and average their consistency scores to reduce the impact of individual model hallucinations; the cost is increased computation, but this is offset by improved robustness. The variance term and downweighting further guard against unreliable descriptions. The calibration parameters are learned per tool separately, which adds memory but accounts for each tool's unique miscalibration pattern; a shared parameter would be more efficient but would ignore tool-specific calibration errors.

### (D) Why it measures what we claim
**Consistency score c_i measures the probability of correctness conditioned on D being faithful. Assumption A: D is a complete and accurate description. Failure mode F: VLM hallucinations or omissions cause consistency to be an unreliable indicator.** In such cases, c_i reflects description error rather than tool miscalibration. **The logistic mapping from c_i and v_i to s_i'** measures **expected accuracy conditional on consistency and description uncertainty** because we assume that consistency is a sufficient statistic for reliability under the given tool and video distribution, and variance captures epistemic uncertainty; this assumption fails when other factors (e.g., temporal context) affect accuracy independently of consistency, in which case s_i' may be biased. The **approximate correctness label** (rouge-L overlap with any description) measures **surrogate correctness** because we assume that textual overlap is a proxy for correctness; this fails when the tool predicts a synonym not present in the description (false negative) or when the description contains irrelevant words (false positive). Despite these assumptions, the cross-modal signal provides a label-free calibration anchor that breaks the reliance on manual calibration data.

## Contribution

(1) A self-supervised calibration framework, CM³, that uses cross-modal consistency between visual tool outputs and textual descriptions to recalibrate confidence scores without labeled calibration data. (2) A demonstration that consistency to an automatically generated description serves as a viable proxy for calibration error, enabling robust tool orchestration under domain shifts. (3) An analysis of the assumptions underlying consistency-based calibration and their failure modes.

## Experiment

### Evaluation Setup

| Role | Choice | Rationale |
|------|--------|-----------|
| Dataset | Ego4D (tool-use subset) | Contains diverse tool-use videos. |
| Primary metric | Expected Calibration Error | Measures confidence calibration directly. |
| Baseline 1 | Uncalibrated raw confidences | Shows need for calibration. |
| Baseline 2 | Temperature Scaling (supervised) | Upper bound with labeled calibration. |
| Ablation-of-ours | Global logistic (shared params) | Tests per-tool adaptation importance. |

### Why this setup validates the claim

This experimental design tests the central claim that cross-modal consistency can replace labeled calibration data. Ego4D's tool-use subset provides realistic video scenes where multiple tools output confidence scores. ECE is the standard metric for calibration quality. Comparing to uncalibrated confidences isolates the calibration effect from other improvements. Comparing to supervised temperature scaling establishes an upper bound and tests whether self-supervised calibration can approach supervised performance. The ablation of replacing per-tool logistic with a global one tests whether per-tool adaptation is necessary. If our method achieves ECE close to supervised scaling, the claim that consistency is a sufficient calibration signal is supported. Conversely, if it fails to improve over uncalibrated, the claim is falsified.

### Expected outcome and causal chain

**vs. Uncalibrated raw confidences** — On a case where the tool predicts a label like "cup" but the video description mentions "mug" (low consistency), the uncalibrated model outputs high confidence and is likely wrong. Our method detects the low consistency and reduces confidence via the logistic mapping, aligning with lower accuracy. Thus we expect a noticeable ECE reduction overall, especially on instances where tool labels and description diverge semantically.

**vs. Temperature Scaling (supervised)** — On a case where the tool label is rare but consistent (e.g., "kettle" matches description), temperature scaling has access to exact ground-truth correctness from a labeled calibration set, so it can directly adjust confidence. Our method relies on an approximate correctness proxy (text overlap), which may be noisy if the description is incomplete. Nevertheless, consistency is a strong indicator. We expect our method's ECE to be slightly higher than temperature scaling on subsets with ambiguous descriptions, but comparable on clear descriptions.

### What would falsify this idea

If our method's ECE improvement over uncalibrated is not statistically significant, or if the gain is uniform across all consistency levels rather than concentrated on low-consistency instances, then the central assumption that consistency reflects accuracy is invalid. Additionally, if the global logistic ablation matches the per-tool variant, the claim that per-tool adaptation is necessary would be false.
