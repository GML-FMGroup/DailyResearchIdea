# Geometric Consistency Check (GCC) for Test-Time Verification of World Model Predictions

## Motivation

Current methods like MindJourney treat world model outputs as perfect, yet inconsistencies across generated views signal failures that degrade spatial reasoning. There is no mechanism to detect or recover from such errors at test time. Specifically, MindJourney's multi-view reasoning assumes all synthesized views are geometrically consistent, but when the world model hallucinates or produces unrealistic views, the VLM aggregates contradictory evidence, leading to incorrect conclusions.

## Key Insight

Under a correct world model, the 3D positions of objects estimated from any two generated views must be consistent via the known camera transformation; geometric inconsistency necessarily indicates a world model failure.

## Method

### (A) What it is
**GCC (Geometric Consistency Check)** is a test-time verification module that takes a set of views generated by a world model and the corresponding camera poses as input, and outputs a binary decision: accept the views as reliable or discard them. It operates without additional training.

### (B) How it works
```python
# Input: views V = {v_1,...,v_n} (images), camera poses P = {p_1,...,p_n} (extrinsics + intrinsics)
# Output: accept (true) or reject (false)

def geometric_consistency_check(V, P):
    # Step 1: For each view, use VLM (Qwen2-VL-7B) to estimate 3D positions with uncertainty
    positions_3d = []
    uncertainties = []
    for v in V:
        # VLM predicts 3D coordinates (x,y,z) and variance (var_x, var_y, var_z)
        pos, var = Qwen2_VL_7B.estimate_3d_position(v)  # returns (x,y,z), (var_x, var_y, var_z)
        positions_3d.append(pos)
        uncertainties.append(var)
    
    # Step 2: Compute pairwise consistency errors with uncertainty weighting
    errors = []
    for i in range(n):
        for j in range(i+1, n):
            # Reprojection error (2D) from view i to view j
            T_i_to_j = get_transformation(P[i], P[j])
            pos_i_proj_3d = T_i_to_j @ positions_3d[i]  # 3D in camera j coordinates
            pos_j_3d = positions_3d[j]
            # 2D pixel error with uncertainty propagation
            reproj_error_2d = (pos_i_proj_3d[0]/pos_i_proj_3d[2] - pos_j_3d[0]/pos_j_3d[2])^2 + (pos_i_proj_3d[1]/pos_i_proj_3d[2] - pos_j_3d[1]/pos_j_3d[2])^2
            var_reproj = compute_var_reproj(uncertainties[i], uncertainties[j], T_i_to_j)  # via Taylor expansion
            # 3D geometric error (distance after transformation)
            geo_error_3d = sum((pos_i_proj_3d[k] - pos_j_3d[k])^2 for k in 0..2)
            var_geo = sum(uncertainties[i][k] + uncertainties[j][k] for k in 0..2)  # simplified
            # Combine into chi-squared statistic with 2+3=5 degrees of freedom
            chi2_stat = reproj_error_2d/var_reproj + geo_error_3d/var_geo
            errors.append(chi2_stat)
    
    # Step 3: Aggregate (mean of chi2 stats)
    mean_chi2 = mean(errors)
    # Step 4: Compare to threshold calibrated on a calibration set
    # Calibration procedure (offline):
    # Gather calibration set of 512 world model outputs that are known correct (synthetic scenes with ground truth)
    # For each sample, compute mean_chi2. Verify normality via Shapiro-Wilk test (p>0.05).
    # Set threshold = chi2.ppf(0.99, df=5)  # 99% confidence level for chi-squared distribution
    threshold =  chi2.ppf(0.99, 5)  # approximately 1.16
    return mean_chi2 < threshold
```

### (C) Why this design
We chose a simple aggregation of pairwise reprojection and geometric errors over a more complex learned consistency classifier because the underlying geometry provides a deterministic, interpretable signal that does not require training data for the verifier. The trade-off is that we rely on the VLM's own 3D estimation accuracy; if the VLM itself is biased, the consistency errors may be inflated even for correct world models, leading to false rejections. To mitigate this, we calibrate the threshold based on a small set of correct world model outputs (512 examples) using a chi-squared distribution, accepting a reduction in sensitivity to subtle failures. We selected reprojection error (2D) and geometric error (3D) jointly because they capture complementary failure modes: reprojection error detects pixel-level misalignment while geometric error detects depth inconsistencies. This adds robustness but increases computational cost. Finally, we use a chi-squared test during calibration instead of percentile-based thresholds because it provides a principled way to control recall without requiring manual tuning; the cost is that the distribution must be approximately chi-squared, which we verify empirically with a Shapiro-Wilk test (p>0.05). Importantly, this method assumes that the VLM's 3D estimates are unbiased and that camera poses are noise-free; violation of these assumptions will inflate the test statistic even for correct world models, leading to false rejections. To reduce sensitivity to VLM bias, we incorporate uncertainty estimates from the VLM (predicted variances) into the test statistic, making the check more robust to estimation noise.

### (D) Why it measures what we claim
The **pairwise reprojection error** measures **geometric consistency** because under correct camera poses and a physically plausible 3D structure, the projection of a point from one view into another should match the observed 2D position in the second view; this equivalence assumes the VLM's 3D estimates are unbiased and the camera poses are accurate. This assumption fails when the VLM systematically misestimates depth or the camera poses are noisy, in which case the error reflects VLM uncertainty or calibration error rather than world model failure. The **pairwise geometric error** (3D distance after transformation) measures **3D structure consistency** because points in a rigid scene should coincide after applying the relative transformation; this assumes the world model produces static scenes without deformation. This assumption fails when the world model synthesizes dynamic content (e.g., moving objects), in which case the error reflects the degree of non-rigidity. The **aggregated mean chi-squared statistic** measures **overall world model reliability** because a correct world model should yield low errors across all view pairs; this assumes that failures are uncorrelated across views. When failures are systematic (e.g., all views are from a hallucinated cat), the mean chi-squared statistic may still be low if the VLM consistently misestimates the same point, and the method may incorrectly accept the views. Sensitivity to such systematic errors is limited without independent verification. The chi-squared formulation, combined with uncertainty weighting, helps to normalize errors by their expected variance, making the test more robust to VLM-specific miscalibration.

## Contribution

(1) A test-time verification algorithm that uses geometric consistency of VLM-predicted 3D positions across views to detect world model failures without additional training. (2) A calibration procedure for the consistency threshold that leverages a small set of correct world model outputs, providing interpretable recall control. (3) An analysis of the failure modes of the consistency check (e.g., rigid vs. non-rigid scenes, VLM bias), guiding practical deployment.

## Experiment

### Evaluation Setup

| Role | Choice | Rationale (≤12 words) |
|------|--------|-----------------------|
| Dataset | SAT benchmark (spatial reasoning) | Multi-view geometry required |
| Primary metric | Answer accuracy after verification | Measures downstream impact |
| Secondary metric | Detection F1 score for geometric inconsistencies | Evaluates verification fidelity |
| Baseline 1 | VLM only (no world model) | Tests necessity of world model |
| Baseline 2 | World model + random verification | Tests benefit of geometric check |
| Baseline 3 | World model + color consistency | Tests importance of 3D geometry |
| Baseline 4 | Oracle (ground truth 3D positions) | Bounds maximum possible gain |
| Ablation-of-ours | GCC without 3D geometric error | Isolates effect of 3D consistency |
| Diagnostic | GCC with ground truth depth (from depth sensor) | Isolates effect of VLM inaccuracies |

### Why this setup validates the claim

This experimental design forms a falsifiable test of the central claim that geometric consistency is a reliable signal for world model verification. The SAT benchmark provides natural multi-view spatial reasoning tasks where geometric consistency is critical. Comparing against a VLM-only baseline reveals whether world model generation overall helps accuracy; comparing against random verification isolates the value of our geometric check; comparing against a naive color-consistency baseline tests the necessity of 3D geometric information. The oracle baseline with ground truth 3D bounds the maximum potential improvement, quantifying the practical headroom. The ablation removes the 3D geometric error term to assess its contribution. The diagnostic using ground truth depth (obtained from a depth sensor during data collection) isolates the impact of VLM 3D estimation inaccuracies on our method's performance. The primary metric—answer accuracy after verification—directly captures the practical benefit of our method. The secondary metric (detection F1) assesses how well GCC identifies geometrically inconsistent world model outputs. If GCC truly detects world model failures as hypothesized, accuracy should improve specifically on cases where geometric inconsistencies are present, while not harming performance on already consistent outputs.

### Expected outcome and causal chain

**vs. VLM only** — On a case where single-view ambiguous spatial relation (e.g., "the cup is behind the bottle" from one view but actually it is to the side), the VLM-only baseline guesses incorrectly because it lacks 3D context. Our method generates multiple views and checks consistency; if the world model produces a consistent 3D understanding, GCC accepts and yields a correct answer. We expect a noticeable accuracy gap on ambiguous spatial relation questions, with minimal gap on simple unambiguous ones.

**vs. World model + random verification** — On a case where the world model hallucinates a non-existent object (e.g., a cat in an empty room), random verification may discard good views or accept bad ones; it often accepts the hallucinated frames, leading to a wrong spatial inference. Our method detects geometric inconsistency (e.g., 3D points of the cat do not align across views) and rejects the entire set, forcing either a regeneration or fallback. We expect our method to show a clear advantage on examples containing hallucinations, but similar performance on clean scenes.

**vs. World model + color consistency** — On a case where the world model produces geometrically distorted but color‑consistent outputs (e.g., an object is scaled incorrectly but retains correct color), color consistency fails to detect the error and accepts the views. Our method's 3D geometric error captures the depth mismatch, leading to rejection. We expect our method to outperform on cases with geometric distortions (e.g., wrong object sizes or positions) while performing similarly on simple cases where color suffices.

**vs. Oracle (ground truth 3D)** — The oracle uses ground truth 3D positions instead of VLM estimates, thus eliminating VLM inaccuracies. We expect the oracle to achieve higher acceptance accuracy and detection F1, especially on cases with subtle geometric inconsistencies that the VLM misestimates. The gap between our method and the oracle quantifies the loss due to VLM estimation errors, providing a bound on potential improvement if VLM were improved.

**Diagnostic (GCC with ground truth depth)** — Replacing VLM's depth estimates with ground truth depth (from a depth sensor) isolates the effect of VLM depth estimation errors. We expect improved performance compared to standard GCC, demonstrating that VLM depth inaccuracies are a source of false rejections. If performance matches the oracle, then the VLM's horizontal positioning (x,y) is sufficiently accurate; otherwise, both depth and x,y contribute.

### What would falsify this idea

If GCC's accuracy improvement is uniform across all question types (both those requiring 3D reasoning and those that do not) rather than concentrated on cases predicted to expose geometric inconsistencies (e.g., ambiguous spatial relations, hallucinations, geometric distortions), then the central claim that geometric consistency is the key verification signal would be falsified. Additionally, if the oracle baseline shows no significant improvement over our method, it would indicate that VLM inaccuracies dominate and geometric consistency is not the limiting factor.
