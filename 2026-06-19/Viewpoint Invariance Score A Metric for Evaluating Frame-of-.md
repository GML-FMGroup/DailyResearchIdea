# Viewpoint Invariance Score: A Metric for Evaluating Frame-of-Reference Agnostic Spatial Reasoning in VLMs with World Models

## Motivation

Existing VLMs exhibit strong spatial bias towards a default frame of reference, as multi-view input alone (e.g., in MindJourney) fails to eliminate this bias because the model's representation remains viewpoint-dependent rather than mapping to a common coordinate frame. This structural limitation persists across architectures, as shown by Qwen2-VL's dynamic resolution improving perception but not addressing viewpoint alignment.

## Key Insight

A canonical top-down view generated by a world model provides a viewpoint-invariant representation because the top-down perspective is invariant to the original camera's azimuth and elevation, collapsing the degrees of freedom that cause bias.

## Method

(A) **What it is**: Viewpoint Invariance Score (VIS) is an evaluation metric that quantifies a VLM's spatial reasoning consistency across different frames-of-reference by comparing answers given raw views versus answers given world model-generated canonical top-down views. Input: VLM (M), world model (W), test scenes each with multiple viewpoints and spatial queries. Output: a scalar VIS ∈ [0,1] measuring residual viewpoint dependency.

(B) **How it works**
```pseudocode
# For each test scene with viewpoints V = {v1,...,vk} and spatial query Q:
for each vi in V:
    c_i = W.generate(vi, condition="top-down")   # generate canonical view
    a_raw_i = M(vi, Q)                            # answer from original view
    a_canon_i = M(c_i, Q)                         # answer from canonical view

# Compute world model consistency score
consistency = agreement_rate({c_1,...,c_k})   # e.g., pairwise LPIPS similarity > 0.9
if consistency < threshold (0.8):
    skip scene or apply calibration factor (e.g., VIS = VIS * consistency)

# Compute inter-viewpoint agreement for raw and canonical sets:
agreement_raw = agreement_rate({a_raw_1,...,a_raw_k})  # e.g., average pairwise accuracy
agreement_canon = agreement_rate({a_canon_1,...,a_canon_k})

VIS = agreement_canon / (1e-6 + agreement_raw)  # normalized measure
```
Hyperparameters: W = Stable Video Diffusion (SVD) with 7 frames, conditioned on camera extrinsics via Plücker embedding; agreement_rate = exact match for multiple-choice queries (e.g., 4-option spatial relations), ROUGE-L for free-form answers; consistency threshold = 0.8 pairwise LPIPS similarity; world model generation batch size = 8, resolution = 256×256.

(C) **Why this design**: We chose agreement across viewpoints rather than absolute correctness to isolate viewpoint invariance from overall reasoning capability; the trade-off is that a model with poor reasoning may still show high agreement if it consistently gives wrong answers. We selected a top-down canonical view over other perspectives (e.g., front-facing) because it is the most viewpoint-agnostic for spatial layout tasks, though this may miss depth-related relationships; the cost is that objects occluded in the original view may not be faithfully rendered from above. We use a world model to hallucinate canonical views from a single input, avoiding the need for multi-view real data, but this introduces dependence on world model fidelity—if the model produces inconsistent layouts across different input viewpoints, VIS may conflate world model quality with VLM invariance; we mitigate this by computing a world model consistency score and filtering out or calibrating scenes where consistency is low. We chose exact agreement as the similarity measure because it is strict and interpretable; it penalizes any variation, but risks penalising synonyms or paraphrases in free-form answers; we mitigate this by using ROUGE-L for non-choice tasks.

(D) **Why it measures what we claim**: The computational quantity `agreement_canon` measures viewpoint-invariant spatial reasoning because, under the assumption that the world model generates a consistent and accurate canonical view from any input viewpoint, a VLM with a truly viewpoint-invariant representation should produce identical answers independent of the original camera position; this assumption fails when the world model hallucinates different layouts from different viewpoints due to occlusions or ambiguous geometry, in which case `agreement_canon` reflects world model consistency rather than VLM invariance—we address this by explicitly measuring world model consistency and filtering out inconsistent scenes. The normalization factor `agreement_raw` measures the baseline viewpoint dependency of the VLM without any canonical transformation; the assumption is that raw inter-viewpoint agreement captures the model's inherent sensitivity to viewpoint, but this fails when the raw views themselves are too similar (e.g., small azimuth changes), so we only use scenes with at least 30° viewpoint variation. The ratio `VIS` measures the relative reduction in viewpoint dependency attributable to canonical view intervention; the assumption is that improvement over raw agreement is due to the canonical representation, but this fails if the world model's rendered views are accidentally easier for the VLM (e.g., fewer clutter), in which case VIS overestimates invariance—we control for this by comparing against a front-facing canonical view baseline.

## Contribution

(1) A novel evaluation metric, Viewpoint Invariance Score (VIS), that quantifies the degree to which a VLM's spatial reasoning is invariant to frame-of-reference by comparing answers from original views versus world model-generated canonical top-down views. (2) A reproducible evaluation protocol that uses existing world models to generate canonical views, enabling assessment of viewpoint invariance without requiring multi-view real data. (3) An open-source dataset of scenes with multiple viewpoints and spatial queries, designed to expose viewpoint bias in VLMs.

## Experiment

### Evaluation Setup

| Role | Choice | Rationale (≤12 words) |
|------|--------|----------------------|
| Dataset 1 | Spatial Action Test (SAT) | Multi-view spatial queries with known ground truth. |
| Dataset 2 | Synthetic3D (proposed, 200 scenes, each 8 views at 45° intervals) | Ground-truth geometry for direct invariance validation. |
| Primary metric | Viewpoint Invariance Score (VIS) | Directly measures residual viewpoint dependency. |
| Baseline 1 | Raw agreement without canonical view | Captures inherent viewpoint sensitivity. |
| Baseline 2 | Direct accuracy averaged over views | Standard performance measure. |
| Baseline 3 | VIS with front-facing canonical view | Alternative canonical perspective. |
| Baseline 4 | Rotation-invariant representation distance (e.g., cosine similarity of ORION features) | Prior invariance metric for comparison. |
| Ablation 1 | VIS with inaccurate world model (e.g., SVD without pose conditioning) | Tests sensitivity to world model quality. |
| Ablation 2 | VIS with ground-truth renderer (Blender, identical scenes) | Isolates VLM invariance from world model. |

### Why this setup validates the claim
The SAT dataset provides diverse viewpoints and spatial queries with ground truth, enabling computation of agreement rates. Synthetic3D adds scenes with known exact geometry, allowing us to compute VIS using a ground-truth renderer as an oracle, which isolates the VLM's viewpoint invariance from world model errors. By comparing VIS with world model vs. ground-truth renderer, we can directly attribute VIS improvements to the canonical transformation. The rotation-invariant representation baseline (Baseline 4) contextualizes our metric against existing work. The front-facing baseline and inaccurate world model ablation further dissect the factors contributing to VIS. If VIS consistently shows higher agreement_canon than agreement_raw on scenes with sufficient viewpoint variation (≥30°) and these improvements align with ground-truth invariance (from synthetic scenes), the claim that VIS measures viewpoint invariance is supported.

### Expected outcome and causal chain

**vs. Raw agreement without canonical view** — On a scene where the VLM gives different answers from two opposite viewpoints (e.g., left vs right of a table), raw agreement would be low (e.g., 0.3). Our method generates a consistent top-down view from both viewpoints, so the VLM's answers become identical, yielding VIS near 1. We expect a large gap (e.g., 0.3 raw vs 0.9 VIS) on such scenes, but close values on scenes with minimal viewpoint variation. On synthetic scenes with ground-truth renderer, the gap should be even larger as no world model inconsistency arises.

**vs. Direct accuracy averaged over views** — Consider a VLM that always guesses "left" regardless of viewpoint. Its accuracy might be 50% on a balanced dataset, but raw agreement would be 1.0 (always same wrong answer). VIS would also be 1.0 if canonical views yield the same wrong answer, incorrectly suggesting perfect invariance. However, our method includes exact match and requires sufficient viewpoint variation; we expect a pattern where high VIS coincides with low accuracy on some scenes, revealing the failure mode (consistent but wrong) rather than true invariance. The ground-truth renderer ablation will disambiguate this: VIS computed with renderer will be low if answers are consistently wrong (since accuracy is low, but if the VLM is consistently wrong even from canonical views, VIS may still be high—this exposes the need for combining VIS with accuracy.)

**vs. VIS with front-facing canonical view** — On a scene where depth ordering matters (e.g., objects behind each other), front-facing views may not disambiguate occlusions. The VLM might give different answers from different front-facing angles. Our top-down view exposes the layout clearly, so VIS should be higher with top-down than with front-facing (e.g., 0.9 vs 0.6). We expect this gap on spatial arrangement tasks, but similar values on tasks where depth is not critical. Synthetic scenes confirm: top-down yields higher VIS than front-facing for depth-sensitive queries.

**vs. Rotation-invariant representation distance** — For a VLM with a poor viewpoint-invariant representation, the cosine similarity of hidden states (Baseline 4) across viewpoints may be moderate (e.g., 0.7). VIS should correlate inversely: high VIS when representation distance is low. We expect a negative correlation (e.g., Pearson r = -0.6) on SAT scenes. However, VIS may be more sensitive because it operates at the answer level, not hidden states.

### What would falsify this idea
If VIS shows improvement equally across all scenes regardless of viewpoint variation magnitude, or if VIS correlates negatively with ground-truth invariance (e.g., from synthetic scenes where we know the layout), or if the world model consistency filter does not remove scenes with inconsistent canonical views, then the central claim that VIS measures viewpoint invariance is falsified. A failure to find a significant correlation with downstream spatial reasoning accuracy would also weaken the metric's usefulness.
