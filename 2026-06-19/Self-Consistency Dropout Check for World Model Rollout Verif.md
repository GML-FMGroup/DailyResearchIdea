# Self-Consistency Dropout Check for World Model Rollout Verification

## Motivation

Existing test-time scaling frameworks like MindJourney rely on world model-generated views for VLM reasoning but lack any mechanism to verify consistency across views. When the world model produces geometrically inconsistent views (e.g., due to out-of-distribution trajectories or ambiguous depth), the VLM's multi-view reasoning degrades without detection. The root cause is that world models are treated as black-box renderers with no self-verification: their outputs are accepted blindly, and no structural property of the generation process is exploited to flag failures.

## Key Insight

A consistent world model should produce the same view regardless of which subset of other views is provided as conditioning—variance across dropout-conditioned regenerations quantifies inconsistency without external priors.

## Method

### (A) What it is
Self-Consistency Dropout Check (SCDC) is a drop-in verification module for world model rollouts. It takes a trajectory of views V = {v_1,...,v_n} generated by a world model WM and returns a per-view consistency score σ_i. High σ_i triggers re-generation or VLM fallback.

### (B) How it works
```python
# Input: WM (world model, supports conditional generation),
#        V = [v_1,...,v_n] (trajectory views),
#        K = 10 (dropout rounds),
#        p_keep = 0.5 (probability to keep a view in condition set)
# Output: consistency_score (scalar, lower = more consistent)

for i in 1..n:
    variances = []
    for k in 1..K:
        mask = [random() < p_keep for _ in 1..n]  # dropout mask, excludes i
        condition_set = {v_j for j in 1..n if mask[j] and j != i}
        v_i_gen = WM.generate(condition=condition_set, target_index=i)
        # v_i_gen is generated view at position i
        store v_i_gen
    # compute pixel-wise variance across K regenerated views
    Var_i = mean_{pixels} var_{k}(v_i_gen^{(k)}[pixel])
    append Var_i
# Aggregate: average variance across views
consistency_score = mean(Var_i)
```

**Hyperparameters**: K=10, p_keep=0.5, pixel-wise variance (no additional feature extractor).

### (C) Why this design
We chose dropout over a fixed held-out set because random subsets probe consistency under different partial observability conditions, exposing failures that might be masked when the same views are always omitted; the trade-off is that K must be large enough to cover diverse masks, increasing compute. We chose pixel-wise variance over feature-level (e.g., LPIPS) to avoid introducing a second model's biases and to keep the check self-contained; the cost is that pixel variance may capture low-level noise (e.g., jitter) rather than semantic inconsistency—acceptable because our target world models (video diffusion) are trained to produce sharp frames, so high variance is dominated by structural differences. We set p_keep=0.5 to balance conditioning information and dropout strength: too high (e.g., 0.9) yields nearly identical conditions, reducing discriminative power; too low (0.1) makes regeneration too ill-posed, causing universally high variance. This design does not collapse onto a simple 'input dropout' baseline because the regeneration is not a prediction task but a consistency test using the same generative model as verifier—a structural coupling prior work (e.g., MindJourney) treats as separate (render vs. reason).

### (D) Why it measures what we claim
**Computational quantity**: pixel-wise variance of regenerated views across dropout masks. This measures **inconsistency** because a geometrically consistent world model should produce the same view regardless of which subset of other views is provided as conditioning—i.e., the model's conditional distribution should be invariant to the context set. The assumption is that the world model's internal representation of the 3D scene is coherent enough that any subset of views uniquely determines the missing view; this assumption fails when the model relies on spurious correlations (e.g., a view that always appears after a certain camera angle regardless of geometry). In that failure case, variance will be low even if the model is inconsistent—to mitigate, we also check variance across dropout rounds with different random seeds (not shown) to expose stochastic collapse. For each view i, the generation conditioned on different subsets is compared: if the model has truly learned 3D structure, all regenerations converge to the same image; if it hallucinates, different cues lead to different outputs. Thus, variance operationalizes the motivation-level concept of 'inconsistency' under the named assumption that the model is conditionally deterministic given perfect geometry; when this assumption fails, variance reflects context sensitivity, which remains a useful proxy for inconsistency.

## Contribution

(1) A self-consistency verification method (SCDC) that uses dropout-conditioned regeneration to detect geometric inconsistency in world model rollouts without external models or 3D priors. (2) A design principle that world models can be used as their own verifiers by exploiting invariance to conditioning subsets, enabling a fail-soft mechanism for test-time scaling frameworks. (3) A plug-and-play module that integrates into existing pipelines (e.g., MindJourney) with negligible architectural changes.

## Experiment

### Evaluation Setup

| Role | Choice | Rationale |
|------|--------|-----------|
| Dataset | SAT spatial reasoning benchmark | Tests geometric consistency in language |
| Primary metric | VLM accuracy after SCDC-filtered rollouts | Directly measures end-task improvement |
| Baseline 1 | Original VLM without world model | Baseline without any world model |
| Baseline 2 | MindJourney (world model, no SCDC) | Tests value of consistency check |
| Ablation of ours | SCDC with LPIPS variance | Checks if pixel variance is sufficient |

### Why this setup validates the claim

The SAT benchmark requires solving spatial reasoning questions that demand consistent 3D understanding. By comparing against a VLM without a world model, we isolate the benefit of generative scene reconstruction. The MindJourney baseline includes a world model but lacks consistency verification, allowing us to measure the incremental improvement from SCDC. The chosen metric, end-task accuracy, directly reflects the practical utility of filtered rollouts. The ablation with LPIPS variance tests whether our simpler pixel-wise variance captures the same consistency signal as a semantic metric. Together, these choices form a controlled test: if SCDC truly identifies inconsistent rollouts, then filtered rollouts should yield higher accuracy, especially on questions where geometric consistency is critical. This setup is falsifiable because we can pinpoint the exact failure mode each baseline should exhibit.

### Expected outcome and causal chain

**vs. Original VLM without world model** — On a spatial reasoning question like "Is object A left of object B?" from a description, the VLM may answer incorrectly due to lack of 3D representation. Our method generates multiple views via a world model and applies SCDC to filter inconsistent rollouts, providing coherent geometry. We expect a noticeable accuracy improvement of ~15% absolute on questions requiring mental rotation or occlusion, with parity on simple ones.

**vs. MindJourney** — On an ambiguous scene where the world model hallucinates (e.g., a floating table due to conflicting cues), MindJourney accepts the inconsistent view because it lacks a consistency check. SCDC regenerates that view with different condition masks; high pixel variance flags inconsistency, triggering re-generation. Thus our method avoids the hallucination. We expect a 20% relative error reduction on questions with conflicting visual cues, while performance matches on obviously consistent scenes.

### What would falsify this idea

If our method does not show a concentrated improvement on questions predicted to require geometric consistency (e.g., those with viewpoint changes or occlusion), or if the gain is uniform across all question types, then SCDC's variance does not specifically measure inconsistency as claimed.
