# Cycle-consistent Layout Refinement for Structure-aware Text-to-Image Generation

## Motivation

Existing methods like IV-CoT rely on expensive sketch annotations and a single forward pass, preventing error correction in complex scenes. This one-pass assumption persists across all branches (e.g., Janus-Pro, GoT-R1), causing structural inaccuracies in multi-object compositions. A feedback loop comparing text-predicted and image-predicted layouts could enable self-correction without external supervision, but no method currently exploits such cycle consistency.

## Key Insight

Cycle consistency between the structural plan predicted from text and the plan reconstructed from the generated image provides a self-supervised training signal, enabling iterative refinement of the plan without ground-truth sketches.

## Method

**CyclePlan**

(A) **What it is:** CyclePlan is a self-supervised framework for text-to-image generation that iteratively refines the latent structural plan by enforcing cycle consistency. Input: text prompt; pretrained text-to-structural-queries encoder (from IV-CoT, a transformer encoder with output dim 512); pretrained object detector (DETR with ResNet-50 backbone, frozen, confidence threshold 0.3); layout-to-structural-queries encoder (2-layer MLP, hidden dim 256, GeLU activation, output dim 512). Output: final image after iterative refinement.

(B) **How it works:**
```python
def CyclePlan(text, num_iterations=3, lr=0.1):
    # Initialize structural queries from text
    S = TextToStructuralQueries(text)  # latent vector (d=512)
    
    for t in range(num_iterations):
        # Generate image with current S (semantic queries fixed)
        I = RenderAppearance(S, fixed_semantic_queries)
        
        # Extract layout from I using frozen object detector
        boxes, labels = PretrainedDetector(I, conf_threshold=0.3)  # set of (x1,y1,x2,y2,class)
        
        # Map layout back to structural queries via learned encoder
        S_pred = LayoutToStructuralQueries(boxes, labels)  # same d
        
        # Cycle loss (MSE)
        loss = torch.nn.MSELoss()(S, S_pred)
        
        # Refine S by gradient descent on loss (S is a parameter)
        S = S - lr * torch.autograd.grad(loss, S, retain_graph=True)[0]
    
    # Final generation
    I_final = RenderAppearance(S, fixed_semantic_queries)
    return I_final
```
Hyperparameters: num_iterations=3, learning rate=0.1, S dimension=512. Each iteration runtime: DETR ~0.15s, diffusion renderer ~0.5s, gradient update ~0.05s; total ~2.1s per image on one A100. Training layout encoder: 10,000 synthetic examples (generated by perturbing S from COCO captions) takes 2 hours on a single A100.

(C) **Why this design:** We chose gradient-based refinement over a learned corrector (which could be faster) because gradient descent directly minimizes the cycle loss, ensuring progress; a learned corrector might overfit to training distribution. We chose 3 iterations as a trade-off between refinement quality (more iterations risk overshooting) and inference speed (each iteration runs detector and rendering). We chose MSE loss over cosine loss because MSE penalizes magnitude differences, which aligns with the goal of achieving identical structural plans; however, MSE is sensitive to scaling. The frozen detector was chosen to avoid training a separate layout predictor, accepting that detection errors may cause noisy gradient signals. The layout-to-structural-queries encoder is trained end-to-end with cycle loss on synthetic data (perturbed queries and generated images) to map detected layouts back to the latent space.

(D) **Why it measures what we claim:** The cycle loss ℒ = MSE(S, S_pred) measures **structural consistency** between the text-predicted plan and the plan recovered from the generated image. **Core assumption:** The frozen object detector (DETR with confidence threshold 0.3) reliably extracts all relevant objects with correct spatial relations from any generated image, and the learned layout encoder (trained on synthetic data) accurately maps those detections back to the same latent structural query space such that MSE(S, S_pred) is minimized only when the image layout matches the text. This assumption is validated by constructing a calibration set of 500 generated images with human-annotated layouts; we report detection recall (target >85%) and precision (>90%) on this set. **Failure mode:** When the detector misses objects (recall <80%) or the encoder is non-injective (two different layouts map to similar S_pred), the cycle loss no longer reflects structural consistency and may corrupt refinement. In such cases, we rely on the confidence threshold to discard low-confidence detections; systematic class misses degrade performance. We quantify this degradation in experiments by adding synthetic occlusions to generated images before detection. The gradient-based update operationalizes **self-correction**: by moving S toward S_pred, we align the latent plan with the image layout, iteratively reducing the discrepancy. The number of iterations controls how aggressively we enforce consistency, balancing accuracy and stability.

## Contribution

(1) A self-supervised cycle-consistency framework for iterative refinement of structural plans in text-to-image generation, eliminating the need for sketch supervision. (2) A concrete algorithm that uses gradient descent on the cycle loss to update latent structural queries, enabling error correction during inference. (3) The insight that layout consistency between text and image can serve as a training signal, opening the door to unsupervised structure learning.

## Experiment

### Evaluation Setup

| Role | Choice | Rationale (≤12 words) |
| --- | --- | --- |
| Dataset | T2I-CompBench | Tests spatial and attribute compositionality. |
| Primary metric | Object Accuracy | Measures correct object presence and relations. |
| Baseline 1 | IV-CoT | Direct predecessor; tests need for cycle consistency. |
| Baseline 2 | Janus-Pro | Strong unified model; tests advantage of self-correction. |
| Baseline 3 | GoT-R1 | Reasoning-based generation; tests latent plan refinement. |
| Ablation-of-ours | CyclePlan w/o iterations | Removes iterative refinement to isolate its effect. |

### Why this setup validates the claim

This setup tests the central claim that iterative cycle-consistency refinement improves structural alignment in text-to-image generation. T2I-CompBench provides fine-grained instances with multiple objects and spatial constraints, enabling detection of structural errors. The primary metric, Object Accuracy, directly quantifies whether generated images contain all specified objects in correct relations. Among baselines, IV-CoT uses a single-pass structural plan but no refinement, isolating the effect of iteration. Janus-Pro lacks explicit structural planning, testing whether a unified model compensates without cycle loss. GoT-R1 uses reasoning chains but not latent plan consistency, probing the role of self-correction. The ablation (no iterations) establishes the baseline performance of the single-shot variant, attributing gains to refinement rather than the encoder design. Together, these comparisons isolate the mechanism of cycle-based refinement from alternative architectural choices. We also include a failure analysis by adding synthetic occlusions to generated images before detection—testing robustness to detector noise. Compute budget: each experiment run (all baselines and ablated variants) on T2I-CompBench (full set) requires ~48 hours on a single A100, using a calibration set of 500 human-annotated images to validate detector accuracy.

### Expected outcome and causal chain

**vs. IV-CoT** — On a case where the initial structural plan misplaces an object (e.g., “a red cube left of a blue sphere”), IV-CoT directly renders that wrong layout because it has no feedback. Our method detects the inconsistency via cycle loss and adjusts S so the object detector output matches the text, yielding correct spatial relations. We expect a noticeable gap on spatial-relation subsets (e.g., 10–15% higher Object Accuracy), but parity on simple single-object prompts where initial plan is already accurate.

**vs. Janus-Pro** — On a case requiring precise count and arrangement (e.g., “three cats and two dogs, all facing left”), Janus-Pro often omits or duplicates entities because it lacks a structured latent plan. Our method’s cycle loss enforces that the detector sees exactly the right objects in the right positions, correcting miscounts. We expect a clear advantage on high-complexity prompts (e.g., 20% higher accuracy on 4+ objects) but smaller gains on simple instructions where Janus-Pro’s implicit reasoning already works well.

**vs. GoT-R1** — On a case with ambiguous spatial relations (e.g., “the cup is on the table but next to the laptop”), GoT-R1 may generate a plausible but incorrect arrangement because its reasoning template does not enforce visual consistency. Our cycle-based refinement uses the detector layout as a concrete target, resolving ambiguity by aligning S with the image content. We predict our method excels on prompts involving multiple relations (e.g., 15% higher accuracy on spatial-relation subsets), while GoT-R1 may outperform on compositional prompts with clear logical steps.

Additionally, under simulated detection noise (e.g., additive Gaussian blur before detection or random erasing of small objects), we expect our method’s advantage to shrink relative to scenarios with clean detections; we quantify this degradation to bound the claim’s scope.

### What would falsify this idea

If CyclePlan shows no significant improvement over IV-CoT or its single-iteration ablation on the spatial-relation subsets of T2I-CompBench, or if its gains are uniform across all prompt complexities rather than concentrated on cases where initial plans are erroneous, then the central claim that iterative cycle-consistency refinement improves structural alignment is false. Additionally, if the cycle loss does not correlate with human judgment of layout consistency (e.g., Pearson r < 0.5 on a held-out set), the assumption underlying the loss is invalidated.
