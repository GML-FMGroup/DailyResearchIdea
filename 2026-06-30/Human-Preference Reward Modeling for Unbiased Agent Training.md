# Human-Preference Reward Modeling for Unbiased Agent Training

## Motivation

Existing methods like Agents-A1 and TÜLU 3 rely on verifier-based reward signals for agent training, which are only available for tasks with programmatically checkable outcomes. This structural limitation introduces bias toward verifiable aspects and fails to provide reward signals for subjective quality, leaving a persistent gap in reward validity for unverifiable instructions. The root cause is the assumption that verifiable outcomes suffice as reward signals, which ignores human judgment on unverifiable but critical dimensions.

## Key Insight

Human preference judgments inherently integrate both objective correctness and subjective quality, so a reward model trained on such judgments can provide a unified reward signal that remains valid regardless of task verifiability.

## Method

### (A) What it is
Human-Preference Reward Model (HPRM) — a transformer encoder (Llama-7B) with a linear projection head that tokenizes an agent trajectory (sequence of observations, actions, and final outcome) and outputs a scalar reward via a regression head, trained on pairwise human preference comparisons.

### (B) How it works
```python
# Stage 1: Collect human preference pairs
for task in domain set D (6 domains: web navigation, creative writing, customer support, coding, game playing, open-ended dialogue):
    for instruction i in sampled instructions (n=1000 per domain):
        generate two trajectories τ_i^1, τ_i^2 using current policy (Agents-A1)
        human labels: 3 independent judges; y_i = majority vote ∈ {0,1} (0: τ^1 better, 1: τ^2 better)
        store (τ_i^1, τ_i^2, y_i)
# Stage 2: Train reward model
Initialize R_θ = Llama-7B + linear head (single layer, no bias, output dim=1, using [CLS] token representation)
hyperparameters: learning_rate=2e-5, batch_size=16, epochs=3, optimizer=AdamW
for each epoch:
    sample batch of pairs (τ^1, τ^2, y)
    r1 = R_θ(τ^1), r2 = R_θ(τ^2)
    loss = -[y * log σ(r1 - r2) + (1-y) * log σ(r2 - r1)]  # Bradley-Terry
    update θ
# Stage 3: RL fine-tune agent policy π_ϕ (MoE architecture) with overoptimization mitigation
Initialize π_ϕ from Agents-A1 checkpoint
Maintain ensemble of 3 reward models (trained independently with different seeds)
Create calibration set C of 512 held-out preference pairs
for each PPO step (clip_epsilon=0.2, learning_rate=3e-6, γ=0.99, β_kl=0.1):
    sample trajectory τ from π_ϕ
    reward = mean_{i=1..3} R_θ_i(τ) - β_kl * KL(π_ϕ(·|τ) || π_initial(·|τ))
    update π_ϕ using PPO with reward
    # Uncertainty-aware early stopping:
    if step % 100 == 0:
        compute ensemble variance on calibration set C, var_c
        if var_c > 0.05: stop training
```
**Load-bearing assumption (explicit):** The reward model's scalar output remains a valid indicator of human preference for trajectories generated by a policy being optimized against it (i.e., no distribution shift or reward overoptimization). Mitigation: We apply KL penalty from the initial policy (β_kl=0.1) and use a 3-model ensemble with uncertainty-aware early stopping (variance threshold=0.05 on calibration set).

### (C) Why this design
[unchanged from original] We chose Bradley-Terry pairwise loss over pointwise regression because it is more robust to label noise and directly optimizes the relative ranking that humans provide, accepting increased computational cost from pairwise sampling. We opted for a transformer encoder over a smaller model to capture long-range dependencies in agent trajectories (average 45K tokens), accepting higher memory usage. Instead of using domain-specific reward models, we train a single model on all domains to encourage transfer and consistency, accepting potential domain specialization trade-off. We used per-trajectory reward rather than per-step reward to reduce human annotation granularity and leverage holistic judgment, accepting that temporal credit assignment is deferred to the RL algorithm. Finally, we initialized the policy from Agents-A1's good checkpoint to accelerate training, accepting potential bias from the original verifier-trained policy. This design contrasts with prior RLVR methods like TÜLU 3, which still require verifiable outcomes; our method removes that requirement entirely by using human preferences as the sole signal.

### (D) Why it measures what we claim
[unchanged from original, plus added mitigation for transitivity] The reward model's scalar output r = R_θ(τ) measures overall human preference alignment for trajectory τ because we train it directly on human pairwise comparisons using the assumption that the Bradley-Terry model accurately captures the probability of preference given the true underlying quality; this assumption fails when human preferences are inconsistent or non-transitive, in which case R_θ approximates a noisy average preference. The pairwise loss σ(r1 - r2) operationalizes relative quality comparison—it measures the relative difference in alignment because the logistic function maps the difference to preference probability; this measure equates alignment if humans' judgments are transitive and consistent, but fails when context effects alter preferences (e.g., order effects) in which case the reward model reflects a biased aggregate. Mitigation: we report Cohen's kappa inter-annotator agreement on the training pairs (target >0.6) and perform consistency checks by measuring transitive closure violations; if violations exceed 5%, we flag the domain. The shared architecture across domains measures unified preference because we assume that a single reward model can capture common aspects of quality across domains; this assumption fails when domains require completely orthogonal quality criteria, in which case the reward model may underfit domain-specific nuances.

## Contribution

(1) We introduce Human-Preference Reward Modeling (HPRM), a method that trains a single reward model on pairwise human comparisons of agent trajectories across multiple domains, replacing domain-specific verifiers and enabling reward signals for unverifiable aspects. (2) We demonstrate that HPRM provides unbiased reward signals for both verifiable and unverifiable instruction types, addressing the structural limitation of verifier-based rewards. (3) We release a human preference dataset for long-horizon agent trajectories spanning six domains.

## Experiment

### Evaluation Setup

| Role | Choice | Rationale (≤12 words) |
|------|--------|------------------------|
| Dataset | Long-horizon agentic trajectories from 6 domains | Covers diverse tasks with long context |
| Primary metric | Win rate vs reference policy in human eval | Directly measures alignment improvement |
| Baseline 1 | RLVR with verifiable outcomes | Compares to verifiable-signal methods |
| Baseline 2 | DPO on same preference pairs | Tests advantage of separate reward model |
| Baseline 3 | SFT on human demonstrations | Standard supervised alignment baseline |
| Ablation-of-ours | HPRM with LSTM encoder | Tests need for transformer's long-context |

### Why this setup validates the claim
This experimental design tests the core claim that human preferences can replace verifiable outcomes for alignment in long-horizon agentic tasks. The dataset's long trajectories (45K tokens) challenge models to capture holistic quality, making the transformer encoder crucial. The baselines isolate key factors: RLVR tests the necessity of verifiable signals; DPO tests the benefit of a separate reward model over direct policy optimization; SFT tests whether preference-based training improves beyond demonstration imitation. The ablation (LSTM) tests whether long-context modeling is essential. The primary metric—win rate in human evaluation—directly measures alignment, the ultimate goal. If our method outperforms RLVR on non-verifiable tasks and outperforms DPO on long-horizon subsets, the claim that human preferences suffice and a transformer encoder captures long-range dependencies is supported. Conversely, failure would falsify the idea.

### Expected outcome and causal chain

**vs. RLVR with verifiable outcomes** — On a creative writing task where outcomes are subjective (e.g., story ending), RLVR cannot define a verifiable reward, so its policy drifts randomly, producing incoherent text. Our method uses human preferences that directly judge overall quality, so it learns to write engaging endings. We expect a large gap on non-verifiable domains (e.g., +25% win rate) but parity on verifiable coding tasks (e.g., +2%).

**vs. DPO** — On a multi-step navigation task with delayed reward (e.g., collect keys to open a door), DPO treats each trajectory independently and may miss that the final reward depends on early steps. Our reward model encodes the entire trajectory, capturing that early key collection leads to success. We expect a noticeable advantage on long-horizon subsets (e.g., +15% win rate) but similar performance on short tasks (e.g., +5%).

**vs. SFT** — On a customer support task where demonstrations show only one polite response, SFT overfits to that style and fails to handle user frustration. Our method uses pairwise comparisons to learn that empathetic responses are preferred, enabling generalization beyond demonstrations. We expect consistent gains across tasks (e.g., +30% win rate).

### What would falsify this idea
If our method achieves comparable win rates to DPO on long-horizon tasks, or if the LSTM ablation matches the transformer performance, then the claims that a separate reward model and long-context encoder are necessary for alignment without verifiable outcomes are falsified.
