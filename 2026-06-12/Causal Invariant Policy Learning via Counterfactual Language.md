# Causal Invariant Policy Learning via Counterfactual Language Augmentation for Vision-Language-Action Models

## Motivation

Current VLA models, as demonstrated by the LIBERO-PRO benchmark, collapse under domain shifts because they memorize spurious correlations between task-irrelevant features and actions, rather than learning invariant causal structures. For instance, state-of-the-art models like CogACT and RT-X drop from over 90% to 0% success under perturbations, revealing that imitation learning on large datasets alone fails to generalize. This structural failure stems from treating all observed correlations as causal, without distinguishing invariant task-relevant features from spurious ones, a meta-gap that persists across multiple VLA architectures and training paradigms.

## Key Insight

Language instructions are compositional interventions that can systematically vary task-irrelevant attributes while preserving the task goal, providing a natural supervisory signal for learning invariant representations through action consistency across linguistic counterfactuals.

## Method

### Causal Invariant Policy Learning (CIPL)

**Assumption**: LLM-generated counterfactual instructions preserve the same task goal and only vary task-irrelevant attributes. We verify this using an automatic entailment check (NLI model).

**(A) What it is**: CIPL augments the dataset with counterfactual language instructions generated by a large language model (LLM) and enforces action consistency across these variations. The input is a base dataset of (image, instruction, action) tuples; output is a policy invariant to task-irrelevant linguistic and visual perturbations.

**(B) How it works**:
```
Input: Base dataset D = {(image, instr, action)}; LLM (GPT-4, temperature=0.7); NLI model (DeBERTa-large-MNLI)
1. For each (img, instr, act) in D:
   a. Prompt LLM: "Generate 3 alternative instructions for the same task that describe the same goal but change task-irrelevant details (e.g., object color, shape, position, phrasing)."
   b. Collect generated instructions instr_1, instr_2, instr_3.
   c. Filter using NLI: keep only instructions where NLI(instr_i, instr) == entailment (discard rate ≈10%).
   d. Create augmented pairs: {(img, instr, act), (img, instr_1, act), ..., (img, instr_3, act)}.
2. Define policy π: (image, instruction) -> action using OpenVLA (7B params) fine-tuned with LoRA (rank=16). Image encoder: SigLIP-L (patch size 16), text encoder: T5-XXL, action head: 2-layer MLP (hidden=512, GeLU).
3. Train π with:
   a. Standard imitation loss: L_BC = MSE(π(img,instr), act)
   b. Consistency loss: L_cons = MSE(π(img,instr_i), π(img,instr_j)) for all i,j in same group.
4. Hyperparameters: K=3; λ_cons=0.5; learning rate 1e-4, linear warmup 500 steps, cosine decay; batch size 64; optimizer AdamW (weight decay 0.01); epochs=20.
5. Hardware: 8 A100-80GB GPUs, training time ~48 hours. LLM API calls: ~90k (30k base instructions × 3).
Output: Trained invariant policy.
```

**(C) Why this design**: We chose LLM-based counterfactual generation over manual augmentation or rule-based templates because LLMs can produce diverse and semantically valid variations covering a wide range of task-irrelevant attributes without requiring domain-specific templates. The trade-off is computational cost and potential invalid instructions—we mitigate with an NLI filter and validation checks (e.g., token length similarity). We chose MSE consistency loss over contrastive or distributional objectives because it directly penalizes output differences and is simpler; for deterministic actions this is sufficient. λ_cons=0.5 balances BC and invariance based on a small validation set; higher weight may cause action collapse, lower weight may not enforce invariance. Compared to image augmentation, our approach leverages language as a causal intervention tool, directly targeting the linguistic spurious correlations that LIBERO-PRO shows are exploited.

**(D) Why it measures what we claim**: L_cons = MSE(π(img,instr_i), π(img,instr_j)) measures invariant action prediction because it penalizes differences when the instruction changes only in task-irrelevant ways; the assumption is that two instructions describing the same task goal should produce identical actions from the same observation. This assumption fails when the LLM generates instructions that inadvertently change the goal—we filter with NLI to discard such cases. L_BC = MSE(π(img,instr), act) measures fit to expert demonstrations, grounding the policy. Together they ensure task execution while enforcing invariance. To extend to visual perturbations, we rely on the assumption that task-irrelevant visual features (e.g., object color) are correlated with language descriptions; enforcing language invariance thus encourages ignoring those visual features, which transfers to visual variations. This assumption is tested in our ablation and visual perturbation analysis.

## Contribution

(1) A novel training framework, CIPL, that uses counterfactual language instructions generated by an LLM to enforce action consistency, thereby learning invariant representations for VLA policies. (2) The principle that language's compositional structure can serve as a source of controlled causal interventions for improving policy robustness, validated through the LIBERO-PRO benchmark. (3) A simple yet effective method to integrate invariance objectives into standard imitation learning without requiring changes to model architecture or additional environment interactions.

## Experiment

### Evaluation Setup

| Role | Choice | Rationale (≤12 words) |
|------|--------|------------------------|
| Dataset | LIBERO-PRO (primary), CALVIN, CLIPort | Test robustness across multiple benchmarks. |
| Primary metric | Success rate on perturbed tasks | Directly measures invariance to perturbations. |
| Secondary metric | Counterfactual validity rate (NLI pass %) | Ensures quality of generated counterfactuals. |
| Baseline 1 | OpenVLA | Standard VLA trained without CIPL. |
| Baseline 2 | RT-2 | Pre-trained VLA without invariance training. |
| Baseline 3 | Image-augmented BC | Uses random crop, color jitter but no language invariance. |
| Baseline 4 | Invariant Risk Minimization (IRM) | Causal graph discovery via visual augmentations. |
| Ablation-of-ours | CIPL w/o consistency loss | Isolates consistency loss contribution. |

### Why this setup validates the claim

This setup directly tests the central claim that CIPL induces invariance to task-irrelevant linguistic and visual perturbations across diverse environments. LIBERO-PRO provides controlled perturbations; CALVIN evaluates long-horizon tasks; CLIPort tests object manipulation with distractions. The primary metric—success rate on perturbed tasks—captures robustness. Baselines OpenVLA and RT-2 represent state-of-the-art VLA models without explicit invariance; comparing against them reveals if CIPL's language consistency yields better generalization. Image-augmented BC isolates visual augmentation effects, and IRM tests if language counterfactuals provide unique benefits over purely visual causal discovery. The ablation (CIPL without consistency loss) pinpoints the contribution of the consistency loss. Counterfactual validity rate quantifies the quality of generated augmentations, grounding the method's reliability. Together, these form a falsifiable test: if CIPL outperforms baselines specifically on perturbed tasks, the claim is supported.

### Expected outcome and causal chain

**vs. OpenVLA** — On a case where the instruction changes from "pick up the red cup" to "pick up the cup" (removing color), OpenVLA may fail due to reliance on color words; its action may deviate. Our method, trained with multiple counterfactual instructions, learns that color is irrelevant. We expect OpenVLA's success rate on linguistically perturbed tasks to be significantly lower (e.g., 40% vs 80% for color variations).

**vs. RT-2** — On a case where object color changes (e.g., red to green cup), RT-2 may still be biased by visual appearance and language correlation; it may fail to generalize. Our method uses consistency loss to enforce identical actions for different colors. We expect RT-2 to have high variance across color perturbations (e.g., 30-70%) while our method maintains consistent performance (e.g., 80%).

**vs. Image-augmented BC** — On a case where both language and visual details change (e.g., different phrasing and background color), image augmentation alone does not perturb language, so the policy remains sensitive to wording changes. Our method directly perturbs language and enforces consistency. We expect image-augmented BC to fail on language changes (e.g., 30%) but perform similarly on visual-only changes (e.g., 70%), while ours excels on both (e.g., 80% and 85%).

**vs. IRM** — On a case where visual augmentations (e.g., color jitter) are present, IRM may learn invariant features but does not leverage language counterfactuals. Our method benefits from explicit language interventions that break spurious correlations between task-irrelevant language and actions. We expect IRM to be competitive on visual perturbations (e.g., 75%) but worse on language-only perturbations (e.g., 60%), while our method achieves 80% across both.

**Ablation (w/o consistency loss)** — Without L_cons, the policy is simply behavior cloning on augmented data, which may not enforce invariance. We expect performance on perturbed tasks to drop (e.g., 50% vs 80%), confirming that consistency loss is crucial.

### What would falsify this idea

If our method shows comparable improvement on both language and non-language perturbations, or if the gain is uniform across all subsets, it would indicate that the consistency loss is not specifically targeting language invariance but rather acts as a general regularizer. Additionally, if the ablation without consistency loss performs similarly to the full CIPL, then the consistency loss is not key.
