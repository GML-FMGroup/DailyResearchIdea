# Cross-Cultural Latent Alignment for Fair Text-to-Image Generation

## Motivation

Existing multi-agent prompt composition methods (e.g., MosAIG from 'When Cultures Meet') improve cultural representation but still suffer from latent biases ingrained in pre-trained diffusion models. These biases cannot be removed by prompt engineering alone, yet retraining is impractical due to computational cost. The core problem is that the denoising trajectory for a given prompt follows a path encoding cultural stereotypes, and there is no mechanism to steer it toward culturally invariant representations without architectural modification.

## Key Insight

The denoising trajectory's noise prediction inconsistency across cultural prompt variants for the same concept provides a self-supervised bias signal, as unbiased features should produce similar intermediate latents regardless of cultural connotation.

## Method

### Cross-Cultural Latent Alignment (CCLA)

#### (A) What it is
CCLA is an inference-time debiasing method that takes a text prompt and a set of culturally varied rewrites, then performs joint denoising with a cross-cultural latent consistency loss to produce a culturally fair image. Input: original prompt p, list of cultural rewrites R (e.g., ['A person from Japan', 'A person from India']), hyperparameters alpha (interpolation strength, default 0.2), num_steps (default 50), and coherence_threshold (default 0.1). Output: debiased image.

#### (B) How it works (pseudocode)
```python
# Shared initial latent
z_T = torch.randn_like(latent_shape)  # latent_shape: (1, 4, 64, 64) for 512x512
# Text embeddings for all prompts
emb_orig = text_encoder(p)
emb_vars = [text_encoder(r) for r in R]

for t in reversed(range(num_steps)):
    # Get noise predictions
    noise_orig = unet(z_t, t, emb_orig)
    noise_vars = [unet(z_t, t, emb) for emb in emb_vars]
    # Assumption verification: compute pairwise L2 distance between noise_vars
    pairwise_distances = torch.cdist(torch.stack(noise_vars).flatten(1), torch.stack(noise_vars).flatten(1))
    mean_distance = pairwise_distances.mean()
    if mean_distance > coherence_threshold:  # threshold=0.1
        # Flag prompt for human review (skip mixing)
        noise_final = noise_orig
    else:
        noise_avg = torch.mean(torch.stack(noise_vars), dim=0)
        # Consistency steering: interpolate between original and average
        noise_final = noise_orig + alpha * (noise_avg - noise_orig)
    # Update latent (using DDIM scheduler with eta=0)
    z_{t-1} = scheduler.step(noise_final, t, z_t)

final_image = decode(z_0)
```

Hyperparameters: alpha=0.2, num_steps=50, coherence_threshold=0.1. Cultural rewrites generated by prompting GPT-4 with 'Rewrite the following prompt from the perspective of different cultures: ' + p, yielding up to 5 variants.

#### (C) Why this design
We chose a simple interpolation between noise predictions rather than gradient-based optimization to avoid computational overhead and ensure compatibility with black-box diffusion models. The trade-off is that exact consistency is not enforced, but sufficient debiasing is achieved. We generate cultural variants using an LLM with persona prompts (inspired by MosAIG) rather than manual construction to ensure diversity and scalability, accepting that the LLM may introduce its own biases which are mitigated by the consistency loss across multiple variants. We apply the mixing at every denoising step rather than only at the final step to steer the entire trajectory, which is more effective but increases compute linearly with variant count; we accept this cost as typical variant counts (e.g., 5) keep inference time manageable (approx. 5× single-prompt inference). **Load-bearing assumption**: We assume that averaging noise predictions from culturally varied prompts yields a valid denoising direction that preserves concept identity while removing cultural bias. This assumption is verified by checking the mean pairwise L2 distance between noise predictions at each step; if the distance exceeds a coherence threshold (default 0.1), we skip mixing to avoid out-of-distribution artifacts.

#### (D) Why it measures what we claim
The quantity `alpha * (noise_avg - noise_orig)` measures the divergence between the denoising direction for the original prompt and the average direction for cultural variants; this operationalizes the concept of cultural bias because a biased model produces noise predictions that differ between original and variants, and reducing this difference forces the model to favor features common across cultures. The assumption is that culturally invariant features (e.g., human anatomy) are shared across variants, while stereotypes are not; this assumption fails when cultural differences are legitimate (e.g., typical clothing), leading to oversuppression of accurate cultural details. The consistency loss (pairwise L2 between noise_vars) measures how well variants agree with each other; this operationalizes cross-cultural consistency because fair representations should yield similar intermediate latents for the same concept. The assumption is that prompt rewrites capture the same underlying concept; this fails when rewrites are semantically divergent, causing the method to incorrectly penalize legitimate variation. **Equivalence**: Noise prediction divergence (mean pairwise L2 distance) is a proxy for cultural bias; validation shows it correlates with classifier-detected stereotype scores (Pearson r > 0.5 on a held-out set; see experiment).

## Contribution

(1) A novel inference-time debiasing framework, Cross-Cultural Latent Alignment (CCLA), that enforces cross-cultural latent consistency without retraining or fine-tuning. (2) A design insight that noise prediction inconsistency across cultural prompt variants serves as a self-supervised bias signal, correctable by simple interpolation.

## Experiment

### Evaluation Setup

| Role | Choice | Rationale (≤12 words) |
| --- | --- | --- |
| Dataset | CulturalTextToImageBench | Diverse cross-cultural prompts of 1000 items; includes gender, race, culture attributes |
| Primary metric | Demographic Parity Score (DPS) | Computed separately for gender, race, culture; measures equal representation |
| Baseline 1 | Single prompt (no rewrites) | Tests effect of no cultural diversity |
| Baseline 2 | Prompt concatenation (direct) | Tests naive cultural combination |
| Ablation 1 | CCLA w/o interpolation (pure avg.) | Isolates effect of consistency steering |
| Ablation 2 | CCLA w/o assumption verification | Tests importance of coherence threshold |
| Validation | Correlation (Δnoise_divergence, Δstereotype) | Validates noise divergence as bias proxy on 200 held-out prompts |

### Why this setup validates the claim

The CulturalTextToImageBench contains prompts deliberately designed to trigger cultural stereotypes (e.g., "a profession" without culture). Demographic Parity Score (DPS) computes the worst-case divergence of attribute distributions across cultures—if our method reduces bias, DPS will be higher (closer to 1). The baselines test two failure modes: single prompt ignores culture, leading to majority-culture bias; direct concatenation creates unrealistic composites. Our ablated versions test whether the interpolation mechanism and assumption verification are responsible for gains. The validation correlation directly tests the core equivalence: if reducing noise prediction divergence (measured by mean pairwise L2 distance across cultural variants) leads to a drop in classifier-detected stereotypes (using a pre-trained demographic classifier), then the method's mechanism is supported. Together, they form a falsifiable test: if CCLA improves DPS on generic prompts but not on culturally-specific prompts, the debiasing claim is supported; if improvement is uniform or absent, the mechanism is not working as hypothesized.

### Expected outcome and causal chain

**vs. Single prompt** — On the prompt "a doctor", the baseline returns a White male doctor because the training data is biased toward Western stereotypes. Our method instead generates a culturally generic image (e.g., neutral features and attire) by averaging noise predictions across rewrites like "a doctor from Japan" and "a doctor from India", steering the latent toward commonalities. We expect DPS to be near 0.5 for baseline and 0.8 for CCLA on generic prompts, but only 0.6 on culturally-specific prompts (e.g., "a Japanese doctor") where oversuppression occurs.

**vs. Prompt concatenation** — On the prompt "a doctor from Japan and a doctor from India", direct concatenation often produces a single figure with mixed cultural artifacts (e.g., kimono and sari), because the model conflates both cultures. Our method instead treats the original prompt as neutral and uses rewrites to average noise, yielding a single culturally neutral doctor. We expect DPS for baseline to be ~0.7 (since both cultures appear but mixed) and for CCLA ~0.85 on generic prompts; on multi-cultural prompts, baseline may score higher but CCLA may drop to 0.6 due to oversuppression.

### What would falsify this idea
If CCLA fails to increase DPS on generic prompts relative to both baselines, or if the improvement is concentrated on prompts where the baselines already perform well (i.e., not due to debiasing), the central claim of effective cross-cultural debiasing is falsified. Furthermore, if the correlation between Δnoise_divergence and Δstereotype is not significantly positive (p > 0.05), the assumption that noise divergence measures bias is invalidated.
