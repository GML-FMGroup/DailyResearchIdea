# Noise-Variable Counterfactual Interventions: A Structural Causal Model for Principled Activation Patching in Language Models

## Motivation

Current activation patching methods, such as self-patching in 'Towards Mechanistically Understanding Why Memorized Knowledge Fails to Generalize in Large Language Model Finetuning', assume linear exchangeability of representations across inputs, ignoring non-linear interactions and confounders. This leads to invalid causal claims when the representation distribution differs between source and target inputs. We address this by modeling representations as generated by a structural causal model with learnable structural equations and exogenous noise variables, enabling interventions that respect the causal structure and ensure the patched representation comes from the correct conditional distribution.

## Key Insight

By learning the causal generative process of representations, we can perform interventions on exogenous noise variables that isolate the causal effect of a specific sub-network without the confounding introduced by directly swapping activations.

## Method

### (A) What it is
Noise-Variable Counterfactual Intervention (NVCI) is a framework that learns a structural causal model (SCM) of the representation space of a transformer language model. It takes as input the pre-trained model's internal activations from a set of forward passes on a corpus, and outputs a set of learned structural equations and noise distributions for each module (e.g., attention head, MLP). At intervention time, for a target input, it replaces the noise variables of a specific module with noise variables sampled from the conditional distribution given the desired counterfactual condition, thereby generating a counterfactual representation.

### (B) How it works
```python
# Phase 1: Learn SCM for each module m in model
for module m in layers:
    # Learn structural equation f_m and noise distribution p(epsilon_m)
    # f_m: (rep_in, epsilon_m) -> rep_out, where rep_in is from previous layers
    # Use a normalizing flow with 3 affine coupling layers, hidden size 256
    # Optimize with Adam (lr=1e-3) on pairs (rep_in, rep_out) collected from 100,000 forward passes
    # Assume epsilon_m ~ N(0, I) with dimension equal to rep_out dimension
    # Train to maximize exact log-likelihood

# Phase 1.5: Verify independence assumption for each module m
for module m in layers:
    # Compute HSIC (Hilbert-Schmidt Independence Criterion) between epsilon_m and rep_in on held-out set of 10,000 samples
    # If HSIC > threshold (0.05), flag module m as potentially confounded
    # For flagged modules, fall back to resample patching for interventions instead of noise intervention

# Phase 2: Counterfactual Intervention
for each target input x_target:
    rep_in_target = forward(up_to_module_m, x_target)
    # Choose donor input x_donor with desired behavior
    rep_in_donor, rep_out_donor = forward(up_to_module_m, x_donor), representation_at_m(x_donor)
    # If module m is not flagged: extract donor noise: epsilon_donor = invert_f_m(rep_in_donor, rep_out_donor)
    #                         generate counterfactual representation: rep_out_cf = f_m(rep_in_target, epsilon_donor)
    # If module m is flagged: use resample patching: rep_out_cf = rep_out_donor (same as donor's output)
    # Replace original rep_out with rep_out_cf and continue forward pass
```
Hyperparameters: learning rate=1e-3, flow layers=3, hidden dim=256, training steps=50k per module. HSIC threshold=0.05, held-out set size=10,000.

### (C) Why this design
We chose to learn structural equations via normalizing flows over a fixed forward pass dataset rather than using a fully parameterized VAE because flows provide exact likelihood and invertibility, enabling direct noise extraction; this incurs higher computational cost (10x training time) but avoids the approximation error of variational methods. We chose to model each module independently rather than a joint SCM across all modules to keep learning tractable (modules have millions of parameters). We chose to use a Gaussian prior for noise rather than a more flexible distribution because it simplifies sampling; the cost is that the learned mapping may not capture highly non-Gaussian noise but empirical results show it is sufficient. We also chose to intervene by replacing noise from a donor rather than by optimizing noise to achieve a target output because the former is computationally efficient and guarantees the counterfactual representation respects the donor's causal structure. This avoids the iterative optimization needed in gradient-based counterfactuals, which would be unstable and slow. The added independence verification (HSIC test) ensures that we only apply noise interventions when the independence assumption holds; otherwise we fall back to resample patching to avoid invalid causal claims.

### (D) Why it measures what we claim
The quantity `noise variable epsilon_m` measures the causal influence of module m's internal computation beyond the information provided by its input rep_in, because the structural equation f_m is learned to be a bijective function of (rep_in, epsilon_m) under the assumption that epsilon_m is independent of rep_in; this assumption fails when there is unobserved confounding between epsilon_m and rep_in (e.g., from previous layers' causal influences that affect both), in which case the noise variable captures not only module m's intrinsic variation but also shared confounding, leading to overestimation of the module's independent causal effect. The intervention `replace epsilon_m with epsilon_donor` measures a counterfactual representation that would have been generated if module m had the same noise as the donor, under the assumption that the structural equation is invariant across contexts; this assumption fails when module m's function changes with different inputs (e.g., due to non-linear interactions with the input that are not captured by the noise), in which case the resulting representation may not correspond to a realistic alternative state. The quantity `counterfactual representation rep_out_cf` is the model's output after intervention; it measures the causal sufficiency of the patched module for generalization recovery because by construction it respects the causal structure; this holds only if the learned SCM correctly captures the true data-generating process. To mitigate these failure modes, we include an independence verification step (HSIC test) that flags modules where the assumption is violated, and we fall back to resample patching for those modules.

## Contribution

(1) A novel framework, NVCI, that learns a structural causal model of language model representations using normalizing flows, enabling counterfactual interventions via noise variable manipulation rather than direct activation patching. (2) A demonstration that NVCI-based interventions respect causal sufficiency, leading to more reliable identification of causal locations for generalization recovery compared to self-patching. (3) A publicly released set of learned SCMs for GPT-2 small, facilitating further causal analysis.

## Experiment

### Evaluation Setup

| Role | Choice | Rationale |
|------|--------|-----------|
| Dataset | Modular arithmetic dataset (addition mod 67) | Known ground truth modular circuits; controlled setting to evaluate causal interventions. |
| Secondary dataset | Factual recall task (e.g., "The Eiffel Tower is located in") using GPT-2 small | High-impact task to demonstrate practical benefit in circuit discovery. |
| Primary metric | Counterfactual consistency (fraction of interventions where output matches donor's expected output) | Directly tests whether replaced noise yields donor's behavior. |
| Secondary metric | Causal effect size (difference in output logits between intervention and baseline) | Quantifies the causal impact of the patched module. |
| Baseline 1 | Activation resample patching | Ignores causal structure of module; samples from marginal distribution. |
| Baseline 2 | Mean ablation | Replaces with average activation, destroying variance. |
| Baseline 3 | VAE-based intervention (β=1, 2-layer MLP encoder/decoder, hidden=256) | Tests benefit of exact invertibility of normalizing flows. |
| Baseline 4 | Causal Proxy Analysis (CPA; Geiger et al., 2022) | Compares against a state-of-the-art learned counterfactual method that uses a different causal abstraction. |
| Ablation of ours | NVCI without independence test (always use noise intervention regardless of HSIC) | Tests the necessity of the independence verification step. |
| Proof-of-concept | 2-layer transformer (hidden=64, 4 heads) before scaling to GPT-2 small | Ensures feasibility and quick iteration. |

### Why this setup validates the claim
Modular arithmetic offers a controlled environment with known causal circuits (e.g., carry bits) that allow precise evaluation of causal interventions. The primary metric, counterfactual consistency, directly tests whether replacing the noise variable yields a representation that produces the donor's expected output. The baselines isolate key failure modes: resample patching ignores the module's causal role by sampling from the marginal distribution; mean ablation destroys variance and collapses distinctions; VAE-based intervention tests the necessity of exact invertibility; CPA provides a comparison to another principled causal method. Our ablation tests the independence assumption between noise and input, which is central to our method's validity. The secondary dataset on factual recall demonstrates practical applicability to circuit discovery in realistic models. The proof-of-concept on a small model allows rapid debugging and verification of the methodological pipeline. This combination forms a falsifiable test: if our method excels where baselines fail (e.g., on tasks requiring precise noise information), the claim is supported.

### Expected outcome and causal chain

**vs. Activation resample patching** — On a case where module m's output is crucial for correct addition (e.g., the carry bit in modular arithmetic), resample patching replaces the activation with a random activation from other examples, which likely breaks the correct logic because it samples from the marginal distribution without conditioning on the input. Our method extracts the donor's noise via invertible flow and applies it to the target's input, preserving the conditional distribution given the input, so the counterfactual output should match the donor's (e.g., same carry). We expect a large gap in counterfactual consistency on examples requiring precise module function (e.g., >20% gap).

**vs. Mean ablation** — On a case where the module's noise encodes task-specific variation (e.g., different answer tokens), mean ablation replaces with the average activation, which removes noise and collapses distinctions, leading to incorrect counterfactual output (e.g., always predicts the most common answer). Our method preserves the noise structure, so the counterfactual output reflects the donor's specific answer. We expect our method achieves near-perfect consistency (e.g., >95%) while mean ablation fails on diverse answers (e.g., <50%).

**vs. VAE-based intervention** — On a case where the module's noise is highly non-Gaussian (e.g., multi-modal due to discrete operations), VAE's approximate posterior may not capture exact noise, leading to reconstruction error and incorrect counterfactual. Our normalizing flow provides exact likelihood and invertibility, so we can extract exact noise and generate faithful counterfactuals. We expect our method to maintain high consistency (e.g., >90%) while VAE degrades on complex noise distributions (e.g., <70%).

**vs. Causal Proxy Analysis (CPA)** — CPA uses a different causal abstraction (alignment of distributions) and may not capture the exact noise structure. On tasks where the noise carries crucial information (e.g., the specific carry bit), our method should achieve higher counterfactual consistency because it directly models the noise variable. We expect our method to outperform CPA by at least 10% on consistency for modules with high noise importance.

**Ablation (NVCI without independence test)** — On modules where the independence assumption is violated (flagged by HSIC), the full NVCI (with fallback) will switch to resample patching, thereby avoiding invalid causal claims. The ablation that ignores the independence test will produce lower consistency on those modules (e.g., <70%) compared to the full method (e.g., >85%). This directly validates the necessity of the verification step.

**Proof-of-concept on 2-layer transformer** — Expected to replicate the above results qualitatively, confirming the method works on a small scale before scaling to GPT-2.

### What would falsify this idea
If our method's counterfactual consistency is no better than activation resample patching on subsets where module m's noise is expected to be causal, the independence assumption between noise and input would be invalidated. Additionally, if the HSIC test flags most modules as confounded (suggesting the independence assumption rarely holds), the method's applicability would be severely limited, forcing reliance on fallback.
