# AutoKB: Unsupervised Knowledge Boundary Discovery for Visual Generators via Output Density Estimation

## Motivation

Existing methods for knowledge boundary discovery, such as SearchGen (Search Beyond What Can Be Taught), rely on curated seed datasets to evaluate generator capabilities, which imposes a supervised bottleneck and limits scalability to new domains. This approach fails structurally because it requires human-annotated examples of both internal and external knowledge for each generator, making it infeasible for open-ended or rapidly evolving domains.

## Key Insight

The generator's internal knowledge distribution can be approximated by the density of its own generated outputs in a pretrained feature space, and retrieval items that significantly deviate from this density are likely external to the generator's knowledge.

## Method

AutoKB is an unsupervised framework that estimates a visual generator's knowledge boundary by fitting a density model to a set of images generated by the generator, and then classifying any candidate retrieval item as internal (known) or external (unknown) based on its likelihood under this density. Input: a generator G, a set of diverse prompts P (automatically sampled from a large caption pool using a determinantal point process to maximize diversity), a retrieval corpus C, and a pretrained feature extractor F (e.g., DINOv2 with 768-dim features). Output: a binary classifier for each item in C indicating whether it falls inside or outside G's knowledge boundary.

(B) **How it works** (pseudocode):
```python
# Step 1: Generate a representative output set from G using diverse prompts
# Use DPP to sample 1000 diverse captions from CC3M (embed with CLIP text encoder, kernel bandwidth=0.5)
prompts = sample_diverse_prompts_DPP(N=1000)
generated_images = [G(p) for p in prompts]

# Step 2: Extract features and estimate density
features = [F(img) for img in generated_images]  # d=768 vectors
# Fit a Gaussian Mixture Model (GMM) with K components (K=5) via EM
# We use a GMM because it is computationally efficient and provides a closed-form density.
density_model = GMM(n_components=5, covariance_type='diag').fit(features)

# Verification: Check density model coverage on held-out generated images (200 images)
heldout_features = [F(G(p)) for p in sample_diverse_prompts_DPP(200)]
ll_heldout = density_model.score_samples(heldout_features)
# Perform Kolmogorov-Smirnov test vs. training log-likelihoods; pass if p>0.05
# If verification fails, increase N or K.

# Step 3: Classify each corpus item
classifier = {}
for item in C:
    feat = F(item)  # item is an image; for text we assume a shared embedding space
    log_likelihood = density_model.score_samples([feat])[0]
    # Threshold tau set as the 5th percentile of log-likelihoods on generated features
    if log_likelihood < tau:
        classifier[item] = 'external'
    else:
        classifier[item] = 'internal'
```
Hyperparameters: N=1000 (number of generated images), K=5 (GMM components), tau=5th percentile of training log-likelihoods, DPP bandwidth=0.5, verification threshold p>0.05.

(C) **Why this design**: We chose a GMM over a kernel density estimator because GMMs scale better to large feature dimensions and large corpus sizes, accepting the cost that a GMM may oversmooth the distribution if the true manifold is non-Gaussian. We chose a diag covariance (instead of full) to reduce parameters and avoid overfitting when N=1000, accepting the cost that it ignores feature correlations. We set the threshold tau as the 5th percentile of training log-likelihoods (rather than a fixed value) because it adapts to the generator's output distribution automatically, but this choice assumes that the generator produces at least 5% of its outputs in a 'typical' region – if the generator is highly diverse, this may misclassify valid internal samples as external. Compared to prior work like SearchGen which requires a curated seed dataset, our method eliminates the need for human annotation by leveraging the generator's own outputs. A domain expert would not describe AutoKB as a variant of SearchGen because SearchGen evaluates generators by comparing against a fixed benchmark, whereas AutoKB learns the boundary from unlabeled generator outputs. The assumption that generator outputs are i.i.d. samples from its internal knowledge manifold is made explicit; to mitigate mode collapse, we use DPP for diverse prompt sampling and add a verification step on held-out samples. If verification fails, we increase N or K.

(D) **Why it measures what we claim**: The feature density p(f) under the GMM measures the generator's 'internal knowledge' because we assume that the generator's outputs are i.i.d. samples from its internal knowledge manifold; this assumption fails when the generator exhibits mode collapse or hallucination, in which case p(f) reflects biases in the generation process rather than true knowledge. The threshold tau (5th percentile) measures the boundary between internal and external knowledge because we assume that external items have feature representations that deviate significantly from the generative distribution; this assumption fails when external items are semantically similar to internal but visually distinct (e.g., a different breed of dog), in which case tau may misclassify them as internal. The GMM log-likelihood quantifies the 'deviation' from internal distribution because we assume the true distribution is well-approximated by a mixture of Gaussians; this assumption fails when the internal manifold is highly non-convex or has holes, in which case log-likelihood may not be monotonic with semantic externalness.

## Contribution

(1) A novel unsupervised framework, AutoKB, that discovers the knowledge boundary of a visual generator without any curated seed datasets, by estimating its output distribution via density estimation. (2) A principled method to classify retrieval items as internal or external based on their feature likelihood under the generator's output density, enabling adaptive retrieval for retrieval-augmented generation. (3) An empirical demonstration that the density-based classifier achieves high accuracy in distinguishing internal vs. external knowledge across multiple generators and domains, validated against the SearchGen-Bench gold standard.

## Experiment

### Evaluation Setup

| Role | Choice | Rationale (≤12 words) |
| --- | --- | --- |
| Dataset (real-world) | SearchGen-Bench (50 prompts, 5k retrieval corpus) | Tests knowledge boundary on diverse prompts. |
| Dataset (synthetic) | Synthetic-KB (generator trained on subset of ImageNet, known boundary) | Provides ground-truth internal/external labels. |
| Primary metric (real-world) | Average CLIP score (ViT-B/32) | Measures prompt-image alignment with retrieval. |
| Primary metric (synthetic) | Classification accuracy (internal vs external) | Directly measures boundary estimation. |
| Baseline 1 | No retrieval | Baseline without external knowledge. |
| Baseline 2 | Naive retrieval (top-1) | Always retrieves, may introduce noise. |
| Baseline 3 | SearchGen (curated seed) | Requires seed annotation, strong prior. |
| Baseline 4 | Oracle (ground-truth boundary on synthetic) | Upper bound for classification. |
| Ablation-of-ours | AutoKB with KDE | Tests GMM density vs kernel density. |

### Why this setup validates the claim

This setup tests the central claim that AutoKB can accurately distinguish internal vs. external knowledge for a generator, thereby enabling selective retrieval that improves generation quality. The real-world dataset (SearchGen-Bench) includes prompts that require external knowledge and others that the generator can handle internally, allowing us to measure the precision of retrieval decisions. The synthetic dataset (Subset-ImageNet) provides a controlled environment with known ground-truth knowledge boundary, enabling direct evaluation of classification accuracy. Comparing against no retrieval shows the benefit of using external knowledge when appropriate, while naive retrieval tests whether blind retrieval harms performance. SearchGen, a supervised method using a curated seed, acts as an upper bound for unsupervised approaches. The KDE ablation isolates the effect of the GMM density estimator. The oracle baseline on synthetic data shows the maximum achievable accuracy. Average CLIP score is chosen for real-world tasks because it directly captures alignment between prompt and generated image, which should improve when relevant external knowledge is incorporated. Classification accuracy on synthetic data directly tests the core operationalization of knowledge boundary via density. If AutoKB correctly identifies external items, we expect higher CLIP scores on prompts that benefit from retrieval, with no degradation on internal prompts, and high accuracy on the synthetic task.

### Expected outcome and causal chain

**vs. No retrieval** — On a prompt requiring external knowledge (e.g., "a rare species of bird not in the generator's training data"), no retrieval produces a generic or distorted bird because the generator lacks specific knowledge. Our method retrieves the highest-likelihood external item (a real image of that bird), inpainting details via a retrieval-augmented pipeline, so the generated image is accurate. We expect a noticeable gap (e.g., +0.05 CLIP score improvement) on external-knowledge prompts but parity on internal prompts.

**vs. Naive retrieval** — On a prompt where the top-1 retrieval is irrelevant (e.g., "a cat" but the corpus returns a dog due to feature similarity), naive retrieval injects misleading visual context, reducing CLIP score. Our method classifies the dog as external (low density under the generator's GMM) and suppresses retrieval, generating a normal cat. We expect our method to outperform naive retrieval on a subset of prompts where the top-1 retrieval is misaligned (e.g., +0.08 CLIP on those cases), with comparable performance otherwise.

**vs. SearchGen** — On a prompt describing a novel concept absent from SearchGen's curated seed dataset (e.g., "a flying car made of glass"), SearchGen fails to retrieve any relevant item because its seed does not cover the concept, resulting in no improvement. AutoKB, using the generator's own outputs to estimate knowledge, can identify the concept as external and retrieve from a larger corpus, leading to a more accurate generation. We expect AutoKB to show a clear advantage (e.g., +0.10 CLIP) on prompts with concepts far from the seed distribution.

**vs. Oracle (synthetic)** — On the synthetic dataset with known boundary, AutoKB should achieve classification accuracy above 80%, while the oracle baseline (perfect boundary) yields 100%. If AutoKB's accuracy falls below 60%, the density assumption is too weak.

### What would falsify this idea

The idea would be falsified if AutoKB's performance is statistically indistinguishable from naive retrieval on external-knowledge prompts, or if the gains are uniform across all prompt subsets (internal and external alike), indicating that the knowledge boundary estimation is not selective. Specifically, if the classification accuracy on the synthetic dataset is ≤55% (near chance), the core claim fails.
