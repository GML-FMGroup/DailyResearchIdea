# NaturalCounterfactual: Attribute-Controlled Counterfactual Image Synthesis for Bias Evaluation in Natural Images

## Motivation

Existing bias evaluation methods such as StylisticBias rely on synthetic faces generated by StyleGAN3, which may not capture real-world diversity or artifacts, limiting the generalizability of findings. TIBET's counterfactual approach achieves controlled single-attribute variation by leveraging a disentangled latent space in synthetic images, a property absent in natural images where attributes are entangled. This structural gap prevents direct application of counterfactual bias evaluation on natural images, necessitating a method that can induce controlled attribute variations while preserving identity in real-world image distributions.

## Key Insight

Fine-tuning a diffusion model on paired natural images that differ only in a target attribute enforces a factored representation where identity is preserved and the attribute change is isolated, enabling controllable counterfactual generation without requiring a disentangled latent space.

## Method

**NaturalCounterfactual: Attribute-Controlled Counterfactual Image Synthesis for Bias Evaluation in Natural Images**

(A) **What it is**: NaturalCounterfactual is a diffusion-based method that takes a natural image and a target attribute change (e.g., add/remove glasses) and outputs a realistic, identity-preserving counterfactual image, enabling controlled single-attribute bias evaluation on natural image datasets.

(B) **How it works** (training and inference pseudocode):
```python
# Training: fine-tune a pre-trained diffusion model (e.g., Stable Diffusion v1.5) on paired natural images
# Dataset: pairs (x_a, x_a') from CelebA, filtered by attribute classifier to ensure only attribute a differs
# Calibration: for each pair, compute residual difference (L2 on VGG features) on non-attribute regions; reject pairs with residual > 0.1 (threshold calibrated on 512 validation pairs)
# Encoder E (frozen), text encoder T (frozen), denoiser UNet UNet with trainable cross-attention for attribute embedding

# Hyperparameters: learning_rate=1e-5, num_steps=10000, batch_size=16, lambda_identity=0.1, lambda_contrast=1.0

for each batch of pairs (x_a, x_a') with attribute labels (a, a'):
    # Encode images to latent z = E(x_a), z' = E(x_a')
    z = E(x_a)
    z' = E(x_a')
    # Add noise to z and z'
    t ~ Uniform(1,1000)
    noise = N(0,I)
    z_t = sqrt(alpha_bar_t)*z + sqrt(1-alpha_bar_t)*noise
    z'_t = sqrt(alpha_bar_t)*z' + sqrt(1-alpha_bar_t)*noise
    # Concatenate attribute embedding e_a = T(a) (e.g., "a person with glasses")
    e_a = T(a)
    e_a' = T(a')
    # Predict noise with UNet conditioned on attribute embedding
    noise_pred = UNet(z_t, t, e_a)
    noise_pred' = UNet(z'_t, t, e_a')
    # Reconstruction loss
    L_rec = MSE(noise_pred, noise) + MSE(noise_pred', noise)
    # Identity preservation: ensure the same identity yields similar latent representations
    # Use ArcFace similarity (cosine) on decoded latents
    x_dec = D(denoised(z_t, noise_pred))  # approximate denoising with 1-step DDIM
    x'_dec = D(denoised(z'_t, noise_pred'))
    L_id = -ArcFace_sim(E_face(x_dec), E_face(x'_dec))  # maximize similarity (negative loss)
    # Total loss
    L = L_rec + lambda_identity * L_id  # note: L_id is negative, so we minimize it
    update UNet + attribute embedding parameters (AdamW, weight_decay=1e-4)

# Inference: given natural image x and target attribute a_target
x_input = x
e_target = T(a_target)
z_noise = sample noise (same size as latent)
# Generate counterfactual by iterative denoising conditioned on e_target
def denoise(z_noise, UNet, e_target, num_steps=50):
    z_t = z_noise
    for t in reversed(range(1, num_steps+1)):
        t_tensor = torch.tensor([t])
        noise_pred = UNet(z_t, t_tensor, e_target)
        z_t = (z_t - beta_t / sqrt(1-alpha_bar_t) * noise_pred) / sqrt(alpha_bar_t)
    return z_t
z_gen = denoise(z_noise, UNet, e_target)
x_gen = D(z_gen)
return x_gen
```

(C) **Why this design**: We chose fine-tuning a pre-trained diffusion model instead of training from scratch because it leverages powerful generative priors, reducing data requirements and training time, at the cost of potential catastrophic forgetting. We opted for a contrastive identity loss over a direct attribute classifier because the former does not require labeled attribute dimensions for all possible variations, avoiding the bias of predefined categories, but it may not enforce strict attribute disentanglement as strongly as a classifier. We decided to use paired natural images rather than synthetic data or unpaired manipulation (e.g., CycleGAN) because paired data guarantees that the only difference between images is the target attribute, isolating the cause effect, but obtaining such pairs is labor-intensive and may limit scalability. This design explicitly addresses the structural limitation of synthetic methods (StylisticBias) by operating directly on real-world distributions, and the absence of latent disentanglement (TIBET) by learning factored representations from paired examples.

(D) **Why it measures what we claim**: The computational quantity of interest is the difference between the generated counterfactual and the original image (e.g., pixel-wise or semantic difference). We claim this difference measures controlled single-attribute variation because the only change introduced by the generative process is the attribute conditioning, assuming the paired training data ensures that spurious correlations are minimized. Specifically, the identity preservation loss L_id measures identity consistency because we assume that cosine similarity of latent features from the same identity should be high under the frozen encoder; this assumption fails when the encoder itself is biased or when the attribute change alters subtle identity cues (e.g., glasses covering eyes), in which case the metric reflects a combination of identity preservation and attribute-induced occlusion. The reconstruction loss L_rec measures attribute-specific generation quality because we assume that the noise prediction error is minimized only when the model correctly models the attribute condition; this assumption fails when the paired images have other hidden differences (e.g., lighting), and then the loss reflects joint variation. To mitigate this, we calibrate pairs using a residual difference threshold (0.1 on VGG features) and exclude pairs exceeding this threshold. Together, the design ensures that attribute changes are isolated only to the extent that the training data is controlled, enabling bias evaluation by comparing model outputs across counterfactual pairs.

## Experiment

### Evaluation Setup

| Role | Choice | Rationale |
|------|--------|-----------|
| Dataset | CelebA paired: 1000 IDs, 2 images per ID (with/without target attribute) filtered by attribute classifier; residual filtering (VGG L2 < 0.1) retains ~800 IDs per attribute | Tests natural image attribute variation while controlling spurious differences |
| Primary metric | Attribute Manipulation Success Rate (AMSR): fraction of generated images where target attribute is present/absent per attribute classifier (ResNet50 trained on CelebA, accuracy >95%) | Quantifies controlled attribute change |
| Secondary metric | Identity preservation: ArcFace cosine similarity between original and counterfactual images | Measures identity consistency |
| Baseline | StylisticBias: evaluates bias via synthetic style variations (StyleGAN3) | Tests synthetic counterpart |
| Baseline | TIBET: identifies biases via latent traversal without counterfactual generation | Tests disentanglement-based approach |
| Ablation-of-ours | Without identity loss (L_id removed) | Isolates effect of identity preservation |

### Why this setup validates the claim

The paired natural dataset with residual filtering ensures that any attribute change is grounded in real-world distributions while minimizing spurious variations. The primary metric AMSR directly quantifies whether the generated counterfactual isolates the target attribute without unintended changes (e.g., lighting or expression shifts). Baselines test different assumptions: StylisticBias uses synthetic style variations that may not capture real-world attribute correlations, and TIBET relies on latent disentanglement without explicit counterfactual generation. The ablation of identity loss tests whether the contrastive mechanism is crucial for preserving identity during attribute manipulation. Together, if our method outperforms baselines on AMSR and the ablation shows a significant drop, it validates that the counterfactual approach enables more controlled bias evaluation. Failure would indicate either insufficient training data or a flawed identity preservation mechanism. The residual filtering calibration (512 validation pairs) ensures that only pairs with negligible non-attribute differences are used, directly addressing the load-bearing assumption.

### Expected outcome and causal chain

**vs. StylisticBias** — On a case where the target attribute is 'adding glasses' to a face, StylisticBias uses style transfer that often distorts other facial features (e.g., skin texture, eye shape) because it lacks identity-attribute disentanglement. Our method uses paired natural images to learn the precise attribute change, so we expect AMSR >90% with identity similarity >0.95, while StylisticBias yields AMSR <70% and identity similarity <0.8.

**vs. TIBET** — On a case where a model correlates 'nurse' with female, TIBET identifies the bias from latent axes but cannot generate a counterfactual of the same identity with a different profession. Our method generates such controlled counterfactuals, enabling per-identity bias measurement. We expect that our method reveals higher variance in bias scores across identities compared to TIBET's aggregated bias estimates.

### What would falsify this idea
If our method's AMSR is not significantly higher than baselines on attributes with strong spurious correlations (e.g., gender+glasses), or if the identity loss ablation shows no performance drop, then the central claim that paired data and identity loss achieve controlled attribute variation is false. Additionally, if residual filtering fails to identify pairs with hidden differences (e.g., expression changes), the method may not isolate the attribute as claimed.
