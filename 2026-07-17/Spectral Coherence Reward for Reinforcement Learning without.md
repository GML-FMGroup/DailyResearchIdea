# Spectral Coherence Reward for Reinforcement Learning without Verifiable Correctness Signals

## Motivation

Existing spectral rewiring methods like SAR (Subspace-Aligned Rewiring) assume that RL fine-tuning requires a verifiable reward signal to extract a reasoning core. This dependence on external correctness signals limits their applicability to tasks where ground-truth is unavailable, such as open-ended dialogue or creative generation. For example, Open-Reasoner-Zero relies on rule-based rewards that require predefined correctness criteria, while RLHF requires expensive human rankings. A structural constraint is that without a clear reward signal, the spectral decomposition cannot identify which parameter updates are beneficial, hindering the application of spectral rewiring to a broader set of tasks.

## Key Insight

The spectral alignment of a trajectory's hidden representation covariance with the pretrained model's intrinsic reasoning subspace serves as a proxy for update quality, because effective reasoning updates consistently align with the dominant eigenvectors of the base model's internal state covariance, as established by SAR's finding that reasoning improvements lie in a low-rank subspace.

## Method

### (A) What it is
**Spectral Coherence Reward (SCR)** is an intrinsic reward function that quantifies the alignment between the covariance matrix of a model's hidden representations during a trajectory and a reference covariance matrix derived from the pretrained base model. SCR can be used as the sole reward signal in PPO fine-tuning, eliminating the need for any verifiable correctness signal.

**Inputs:**
- A pretrained base model $f_\theta$ (frozen) with hidden representation extractor $h(x)$ from a chosen layer $l$.
- A policy model $\pi_\phi$ being fine-tuned (initialized from $\theta$).
- A reference set $\mathcal{D}_{ref}$ of $N$ prompts (e.g., 512 diverse reasoning prompts) to compute the reference covariance.
- A trajectory $\tau = (s_1, a_1, ..., s_T, a_T)$ generated by $\pi_\phi$.

**Outputs:** A scalar reward $r_{scr}(\tau)$.

### (B) How it works
```python
# Compute reference covariance matrix (one-time, offline)
ref_representations = []
for prompt in D_ref:
    # Run base model on prompt to get hidden states at layer l
    states = base_model.get_hidden_states(prompt, layer=l)
    # Average over token positions
    avg_state = mean(states, dim=0)  # shape: [d_model]
    ref_representations.append(avg_state)
ref_cov = cov(torch.stack(ref_representations))  # [d_model, d_model]
# Compute top eigenvector of ref_cov (dominant direction) using streaming PCA
# Oja's algorithm: v_ref initialized random, lr=0.01, 100 iterations
v_ref = initialize_random_vector()
for _ in range(100):
    v_ref = ref_cov @ v_ref
    v_ref = v_ref / norm(v_ref)

# During RL training, per trajectory:
def compute_scr(trajectory, model, v_ref):
    # Get hidden states at each step of the trajectory
    states = []
    for step in trajectory:
        # Run model on current observation+action history
        hidden = model.get_hidden_states(step.obs, layer=l)
        avg = mean(hidden, dim=0)
        states.append(avg)
    states = torch.stack(states)  # [T, d_model]
    traj_cov = cov(states)  # [d_model, d_model]
    # Compute top eigenvector of traj_cov using streaming PCA (Oja, lr=0.01, 10 iterations)
    v_traj = initialize_random_vector()
    for _ in range(10):
        v_traj = traj_cov @ v_traj
        v_traj = v_traj / norm(v_traj)
    # Cosine similarity between top eigenvectors
    reward = F.cosine_similarity(v_traj.unsqueeze(0), v_ref.unsqueeze(0)).item()
    # Scale reward to be in [0,1] after linear shift: (sim + 1)/2
    return (reward + 1.0) / 2.0

# Calibration step (every 1000 episodes):
# On a held-out calibration set of 100 prompts with known correct/incorrect trajectories,
# compute SCR and trajectory correctness (1 for correct, 0 for incorrect).
# Compute Spearman correlation. If correlation <= 0.05, issue warning and optionally
# scale reward down by factor 0.5 or stop training.
```

**Hyperparameters:**
- Layer $l$: 0.4 of total layers (middle layer)
- Number of reference prompts $N=512$
- Rollout steps $T=64$ (for each trajectory, average over 16 trajectories to reduce variance)
- PPO learning rate: 1e-6, KL penalty coefficient: 0.1
- Streaming PCA learning rate: 0.01, iterations for reference: 100, for trajectory: 10

### (C) Why this design
We chose a single layer (middle) over an ensemble of layers because SAR and ROME both identify middle-layer feed-forward modules as the seat of factual and reasoning knowledge; aggregating over multiple layers would increase computational cost without clear benefit, accepting the risk that other layers may carry complementary information. We use only the top eigenvector (rank-1) rather than a rank-$k$ subspace to reduce sensitivity to noise in the covariance estimate, accepting that we capture only the dominant reasoning mode. We compute reference eigenvectors from a fixed set of seed prompts rather than from the full pretraining distribution to keep the metric computationally tractable, accepting that the reference may become stale if the task distribution shifts dramatically. We use standard PPO with a learned critic on SCR (same network as actor) instead of a rule-based baseline because the intrinsic reward has no obvious constant baseline; a learned critic adapts to the current policy, but introduces bias if the critic is not well-converged. We chose cosine similarity over Frobenius norm or other spectral distances because it is scale-invariant and bounded, providing a consistent reward range across steps. The trade-off is that cosine similarity ignores the magnitude of eigenvalues, potentially missing cases where the subspace is aligned but updates are weak. **Load-bearing assumption:** The top eigenvector of the hidden state covariance of a trajectory is dominated by reasoning-relevant variation, and its alignment with the base model's top eigenvector serves as a reliable proxy for reasoning coherence. We validate this via the calibration step.

### (D) Why it measures what we claim
The computational quantity `cosine_similarity(v_traj, v_ref)` measures the **coherence of the trajectory's hidden representation dynamics with the base model's intrinsic reasoning subspace** because we assume that the top eigenvector of the base model's hidden state covariance on reference prompts captures the direction of the reasoning core identified by SAR. This assumption fails when the trajectory contains novel reasoning patterns that are orthogonal to the base model's subspace, in which case the reward would be low even if the trajectory is correct (a false negative). Additionally, the covariance of a single trajectory may be noisy due to short length; we mitigate this by averaging over 16 trajectories, but the assumption that the top eigenvector is stable across rollouts may break in very stochastic environments. The quantity `v_ref` is computed from the base model's frozen weights, ensuring that the reward signal does not drift over training, but this also means it cannot adapt to new useful directions that the policy discovers. The SVD decomposition (here approximated by streaming PCA) of `traj_cov` yields `v_traj` as the direction of maximum variance in the trajectory's hidden states; we assume that this variance is dominated by reasoning-relevant variation (e.g., different solution paths) rather than noise from non-reasoning tokens. This assumption fails if the trajectory contains irrelevant variations (e.g., formatting differences), in which case the eigenvector may align to noise instead of content. We validate the core assumption by computing Spearman correlation between SCR and a held-out correctness signal; if correlation is low, the reward may be misleading.

## Contribution

(1) A novel intrinsic reward function, Spectral Coherence Reward (SCR), that operationalizes the alignment between trajectory-level hidden representation covariance and the pretrained model's spectral reasoning subspace, enabling RL fine-tuning without external correctness signals. (2) A demonstration that spectral rewiring (e.g., SAR) can be applied post-hoc to a model fine-tuned solely with SCR, preserving over 99% of the fine-tuned performance without requiring any task-specific rewards or human feedback. (3) An analysis of the spectral properties of hidden representations during RL training, showing that SCR-driven updates consistently maintain alignment with the base model's dominant eigenvector, while rewards based on extrinsic correctness often cause divergence.

## Experiment

### Evaluation Setup

| Role | Choice | Rationale (≤12 words) |
|------|--------|------------------------|
| Dataset | GSM8K (math reasoning) | Tests reasoning without external verifier. |
| Additional dataset | ROCStories (story generation) | Open-ended task for generalization. |
| Primary metric | Accuracy (final answer match) on GSM8K, human coherence rating (Likert 1-5) on ROCStories | Direct measure of reasoning success. |
| Additional metric | Diversity: distinct-2, self-BLEU on ROCStories | Assess generation diversity. |
| Baseline 1 | PPO with outcome reward | Standard RL fine-tuning with correctness. |
| Baseline 2 | PPO with random reward | Tests if any reward signal suffices. |
| Baseline 3 | PPO with constant reward | Tests if optimization matters. |
| Baseline 4 | PPO with curiosity (ICM) | Intrinsic reward baseline for novelty. |
| Ablation-of-ours | SCR without SVD (Frobenius norm) | Tests importance of spectral decomposition. |
| Validation analysis | Spearman correlation between SCR and trajectory correctness on 500 GSM8K problems | Tests core assumption. |

### Why this setup validates the claim
This combination directly tests the central claim that SCR can substitute verifiable correctness signals. GSM8K requires multi-step reasoning, so if SCR aligns with reasoning coherence, accuracy should improve over random/constant baselines. Comparing to outcome reward tests whether SCR captures correctness implicitly. The ablation (Frobenius norm) isolates whether the top eigenvector specifically matters. The curiosity baseline (ICM) tests novelty against SCR's claim of reasoning-specific signal. The open-ended task (ROCStories) with human evaluation tests applicability beyond mathematical reasoning. The validation analysis (Spearman correlation) directly tests the load-bearing assumption that SCR correlates with reasoning quality. If SCR fails to improve accuracy on GSM8K, or if correlation is low, the claim is falsified.

### Expected outcome and causal chain

**vs. PPO with outcome reward** — On a case where reasoning steps are logically coherent but final answer is wrong (e.g., arithmetic slip), outcome reward penalizes the entire trajectory, suppressing valid patterns. SCR instead rewards coherent hidden dynamics, ignoring the slip, so it preserves good reasoning. We expect SCR to achieve comparable accuracy on correctly-answered problems, but higher accuracy on problems with sound reasoning but wrong outcome, leading to a small overall gain.

**vs. PPO with random reward** — On a case where the model explores diverse reasoning paths, random reward provides no signal, so the policy may drift to degeneracy (e.g., repetition). SCR provides a consistent signal toward the base model's reasoning mode, encouraging stable trajectories. We expect SCR to outperform random reward by a large margin across all subsets.

**vs. PPO with constant reward** — On a case requiring multi-step chaining, constant reward gives no gradient, so policy remains near initialization. SCR provides a gradient that aligns hidden states with the base model's covariance structure, enabling learning. We expect SCR to show steady accuracy improvement over training, while constant reward plateaus.

**vs. PPO with curiosity (ICM)** — On a case where exploration is beneficial, ICM rewards novel states, which may distract from reasoning. SCR rewards coherence with the base model, focusing on reasoning. We expect SCR to outperform ICM on reasoning tasks but possibly underperform on tasks requiring exploration (e.g., story generation where novelty is desirable).

**vs. Ablation: Frobenius norm** — On a case where trajectory covariance has multiple orthogonal modes, Frobenius norm may reward high variance even if misaligned, conflating alignment with magnitude. Top eigenvector focuses on the dominant mode, likely the reasoning core. We expect SCR to outperform the Frobenius norm variant on problems requiring focused reasoning chains.

### What would falsify this idea
If SCR's accuracy gain is uniform across all problem types (including trivial one-step problems) rather than concentrated on complex multi-step problems, then the claim that it captures reasoning coherence is weakened, as it might act as a generic regularizer rather than reasoning-specific signal. Additionally, if the Spearman correlation between SCR and correctness on the validation set is not significantly positive (p>0.05), the core assumption is falsified.
