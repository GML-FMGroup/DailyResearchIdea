# Trace2Tree: Inferring Behavior Trees from Dynamic Execution Traces via Minimum Description Length Induction

## Motivation

Existing methods like Harness Handbook construct behavior trees from static code analysis, which fails on obfuscated, dynamically generated, or legacy code where conventional patterns like clear function calls are absent. The root cause is their reliance on static structural cues that are either missing or intentionally obscured in such codebases. To address this meta-level gap, we need a method that bypasses static code analysis entirely and instead induces hierarchical behavior from observable execution traces alone.

## Key Insight

Any program's observable behavior is a sequential process generated by a hierarchical program; by treating execution traces as symbolic sequences and applying grammar induction with a minimum description length (MDL) prior, we can recover the minimal hierarchical program (behavior tree) that compresses the traces best, without needing readable code.

## Method

### (A) What it is
**MDL-BGI (Minimum Description Length Behavioral Grammar Induction)** takes a set of execution traces (sequences of events) as input and outputs a behavior tree (a hierarchical grammar where each nonterminal corresponds to a high-level behavior). It uses an MDL-guided greedy merging algorithm to infer the grammar that best compresses the trace dataset.

### (B) How it works
```python
Input: Set of execution traces T = {t1, t2, ..., tn}, each ti is a sequence of event symbols from alphabet Σ.
Output: Behavior tree (CFG) G with start symbol S, nonterminals N, productions P, and mapping to behavior hierarchy.

Parameters:
- min_occurrence: minimum times a pattern must appear to be considered (default 2)
- max_rule_length: maximum number of symbols in a candidate rule (default 10)

Algorithm:
1. Initialize G with each trace as a production from S, using unique nonterminals for each trace? 
   Actually, start with a flat grammar where each trace is a sequence of terminals. 
   Let P contain a single rule S -> concatenation of ALL traces separated by a separator token $. 
2. Compute initial description length: DL = |G| + sum over traces of encoded length using G.
   (use fixed number of bits per symbol, say log2(|Σ|+|N|) for each symbol, plus bits for rule definitions)
3. Repeat:
   a. For each pair of symbols (A, B) in current grammar (could be terminals or nonterminals),
      compute DL if we add a new nonterminal C -> A B, and replace all occurrences of A B in the grammar with C.
   b. For each possible new rule candidate (a sequence of 2..max_rule_length symbols) that occurs at least min_occurrence times,
      compute DL if we add that rule and replace all occurrences.
   c. Evaluate merges of two existing nonterminals: if all occurrences of pattern P that form one nonterminal and another nonterminal are identical, try merging.
   d. Select the operation that reduces DL the most (or stop if no reduction > threshold).
4. Convert grammar to behavior tree: each nonterminal becomes a node, and its children are the symbols in its right-hand side.
   For nonterminals that represent loops/stars, we can annotate (e.g., via frequency analysis).
5. Output behavior tree G.
```

### (C) Why this design
We chose MDL over simpler frequency-based induction (e.g., Sequitur's rule-of-two) because MDL provides a principled way to balance model complexity against data fit, preventing overfitting on noisy traces; the trade-off is increased computational cost per iteration. We used greedy search over candidate merges rather than exact inference (e.g., Bayesian grammar induction) because exact inference is exponential in the number of symbols, whereas greedy is tractable for large traces; the cost is that we may miss globally optimal grammars, but in practice the greedy sequence often yields interpretable hierarchies. We opted for a fixed symbol encoding (log2 of alphabet size) rather than adaptive coding because it simplifies DL computation and aligns with the minimal tree assumption; the downside is that we do not optimally encode symbols with varying frequency, potentially inflating the cost of rare symbols. Lastly, we limited candidate rule length to 10 to avoid combinatorial explosion of long patterns, accepting that very long repeated subroutines may be missed; however, any behavior tree node should be decomposable into smaller sub-behaviors, so hierarchical decomposition should still capture long routines as nested rules.

### (D) Why it measures what we claim
- **Grammar size (number of rules and total symbol count)** operationalizes *structural complexity* of the behavior tree because it counts the bits needed to describe the hierarchical model, assuming a fixed encoding. This equivalence relies on the assumption that the grammar truly represents the program's hierarchical structure; when the trace contains non-hierarchical control flow (e.g., arbitrary jumps), the grammar may add extra rules to compress flat patterns, thus overestimating structural complexity, and in that case the metric reflects compression efficiency rather than true hierarchy. - **Encoded trace length** operationalizes *how well the grammar explains observed behavior* because it is the number of bits to represent traces given the grammar. This assumes that the grammar is the true generative model; when the traces contain noise (e.g., environment randomness unrelated to program logic), the grammar will need extra rules or symbols to account for it, causing the encoded length to include irreducible noise, so the metric reflects both program behavior and environmental stochasticity. - **MDL change (ΔDL)** operationalizes *goodness of a merge* because a negative ΔDL means the addition of a rule reduces total description length, indicating that the introduced pattern is a genuine compression of behavior. This relies on the assumption that compression corresponds to hierarchical abstraction; when multiple equally good merges compete, the greedy selection may pick a locally optimal but globally suboptimal pattern, causing the final grammar to be larger than necessary, and the metric then reflects a local minimum of description length rather than the global one. - **The mapping from nonterminal to behavior tree node** operationalizes *hierarchical behavior composition* because each nonterminal corresponds to a repeated pattern that we label as a behavior. This assumes that the pattern's occurrences all represent the same high-level behavior; when the same symbol sequence occurs in different contexts with different semantic meanings (e.g., a 'print' action in error handling vs. logging), the grammar conflates them, and the behavior tree node becomes a generic placeholder rather than a specific behavior.

## Contribution

(1) An algorithm (MDL-BGI) that infers behavior trees from dynamic execution traces using minimum description length grammar induction, bypassing static code analysis entirely. ## Experiment

### Evaluation Setup

| Role | Choice | Rationale (≤12 words) |
| ---- | ------ | --------------------- |
| Dataset | Synthetic hierarchical DFA traces | Known ground truth hierarchy. |
| Primary metric | F1 of nonterminal recovery | Directly measures hierarchy accuracy. |
| Baseline 1 | Sequitur algorithm | Frequency-based grammar induction. |
| Baseline 2 | N-gram (order-2) model | Captures local patterns only. |
| Baseline 3 | Random merging | Control for chance grammar structure. |
| Ablation | MDL-BGI without MDL | Frequency-only merging to test MDL. |

### Why this setup validates the claim
Synthetic traces with known ground truth hierarchies allow direct measurement of recovery accuracy via F1 score. Comparing against Sequitur isolates the benefit of MDL's principled trade-off over fixed-frequency rules. The n-gram baseline tests whether local patterns suffice, and random merging controls for chance. The ablation (no MDL) tests whether MDL's complexity penalty is critical. This combination ensures any observed advantage of MDL-BGI is attributable to its MDL-driven merging, not architecture or data artifacts.

### Expected outcome and causal chain

**vs. Sequitur** — On a case where a meaningful pattern appears rarely (e.g., once in a 100-trace dataset), Sequitur's rule-of-two requires at least 2 occurrences, so it fails to infer a rule. Our method evaluates compression gain: even a single occurrence can be worth encoding if it reduces overall description length (e.g., it is a long pattern). We expect MDL-BGI to achieve higher F1 on rare but semantically important patterns, while matching Sequitur on frequent ones.

**vs. N-gram** — On a case where a high-level behavior consists of a long sequence of subactions (e.g., 7-step nested routine), the n-gram model cannot capture nesting beyond order-2, so it produces fragmented, local predictions. Our method builds hierarchical rules, so it recovers the full sequence as a single nonterminal. We expect MDL-BGI to show a significant F1 advantage on traces with long-range hierarchical dependencies, while both perform similarly on flat short sequences.

**vs. Random merging** — On any trace set, random merging produces arbitrary nonterminals that do not correspond to true behaviors, yielding low F1. Our method's greedy MDL search systematically identifies patterns that compress best, so we expect MDL-BGI to achieve substantially higher F1 on all subsets.

### What would falsify this idea
If MDL-BGI's F1 is not significantly higher than baselines specifically on traces with rare or nested patterns (where hierarchy is crucial), then the claim that MDL provides superior hierarchical recovery is falsified.
