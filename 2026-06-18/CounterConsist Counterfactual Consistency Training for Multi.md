# CounterConsist: Counterfactual Consistency Training for Multi-Image Attribute Extraction

## Motivation

Existing multi-image attribute extraction methods, such as those evaluated on IndustryBench-MIPU, often fail to perform effective cross-image reasoning because standard supervised training allows models to memorize spurious correlations from individual images. This structural limitation arises because the training objective does not penalize reliance on a single image, leading to poor generalization when images contain conflicting information. A concrete instance is the performance drop observed in IndustryBench-MIPU when models must aggregate evidence from multiple images, indicating a lack of genuine cross-view reasoning.

## Key Insight

By training on examples where one image is replaced with a conflicting attribute value, the model must detect the inconsistency and infer the correct consensus, enforcing a causal reliance on cross-image comparison rather than spurious single-image cues.

## Method

### (A) What it is
CounterConsist is a counterfactual training procedure for multi-image attribute extraction. It takes a batch of product examples, each with multiple images and a consensus attribute value, and trains a model comprising a shared backbone (ResNet-50, pretrained on ImageNet), a consensus extraction head (transformer with cross-attention, 4 heads, 256 hidden dim), and a per-image consistency verification head (2-layer MLP with hidden dim 128, ReLU). Outputs are trained model parameters.

### (B) How it works
```python
# Hyperparameters: α = 0.5 (consistency loss weight), k = 1 (number of replacements per example), p = 0.3 (replacement probability with conflict)
# Precomputed pool: for each attribute value, a set of 200 images from different products with that same value but other attributes different (real only). For synthetic conflicts, we also include 200 images per attribute generated by a StyleGAN2 trained on subsets of the dataset (only for training, not evaluation). The pool is curated to ensure visual diversity.
# Calibration: After training, we apply temperature scaling (learned temperature τ on a held-out validation set of 512 examples with both real and artificial conflicts) to the consistency verification logits to improve probability calibration.

for batch in D:
    # Original batch
    H_orig = [backbone(images) for images in batch]  # list of feature sets, each with n features (n=4 images per product)
    pred_orig = extraction_head(aggregate(H_orig, use cross-attention across positions))    # consensus prediction
    
    # Counterfactual batch
    batch_cf = []
    for example in batch:
        S = random.sample(range(n), k)
        for i in S:
            if random.random() < p:
                # Replace with conflicting image: same attribute, different value
                example.images[i] = sample_conflicting_image(example.consensus)  # from pool, with 50% chance from real pool, 50% from synthetic pool
            else:
                # Replace with another image from same product (no conflict)
                example.images[i] = random_same_product_image(example)
        batch_cf.append(example)
    H_cf = [backbone(images) for images in batch_cf]
    pred_cf = extraction_head(aggregate(H_cf))
    
    # Consistency verification: per-image binary logit (1=inconsistent)
    scores_cf = [consistency_head(features) for features in H_cf]
    true_labels = generate_inconsistency_labels(batch_cf)  # 1 for replaced images (both conflict and same-product replacements)
    loss_consistency = binary_cross_entropy_with_logits(scores_cf, true_labels)  # standard BCE
    loss_extraction = cross_entropy(pred_orig, true_consensus) + cross_entropy(pred_cf, true_consensus)
    total_loss = loss_extraction + α * loss_consistency
    total_loss.backward()
    optimizer.step()  # Adam, lr=1e-4, weight decay=1e-5, batch size 32
```

### (C) Why this design
We chose to generate counterfactuals by replacing images with conflicting ones from a pre-built pool rather than by perturbing the image itself (e.g., adding noise), because semantic conflicts (different attribute values) directly force the model to compare high-level concepts, not low-level noise. The trade-off is the need for a curated pool, but it ensures the conflict is meaningful. We chose to have separate extraction and consistency heads sharing a backbone instead of a single multi-task head, because the consistency head can specialize on conflict detection without interfering with the extraction head's gradient; the cost is increased parameters and potential optimization imbalance. We chose to apply the consistency loss only on counterfactual examples (not on originals) to avoid the trivial solution of always predicting consistency; this design allows the head to learn the signal of conflict. The hyperparameter α=0.5 balances the two losses based on initial tuning, accepting that the optimal trade-off may vary per dataset. Finally, we train extraction loss on both original and counterfactual batches to maintain performance on standard examples while also forcing correctness under conflict. A key assumption underlying this design is that the consistency verification head, trained on artificially introduced conflicts, will generalize to detect natural inconsistencies in real-world multi-image attribute extraction. We calibrate the head using temperature scaling on a held-out validation set of 512 examples that include both real and artificial conflicts, and we monitor the calibration error (ECE). If the assumption fails (e.g., synthetic artifacts create shortcuts), the calibration will indicate poor reliability, and we will detect via performance on the validation set.

### (D) Why it measures what we claim
The consistency verification head's per-image logit measures whether an image is inconsistent with the consensus because it is trained on supervised labels of artificially introduced conflicts; the assumption is that the model can learn a generalizable notion of attribute consistency that transfers to real-world conflicts. This assumption fails when the counterfactual replacements are not representative of natural inconsistency patterns (e.g., only extreme cases), leading the head to overfit to artificial cues. To mitigate this, we diversity the pool with both real and synthetic conflicts. The extraction head's prediction on counterfactual examples measures the ability to ignore conflicting inputs and infer the correct consensus because the label is the original consensus; the assumption is that the model learns to down-weight conflicting images rather than simply relying on majority non-conflicting images. This assumption fails when the model uses a voting mechanism without explicit conflict resolution, which can still achieve correct predictions but does not require cross-image reasoning. The cross-attention aggregation in the extraction head operationalizes cross-image comparison because attention weights enable each image to reference others; the assumption is that attention correlates with consistency. This assumption fails when attention is dominated by irrelevant visual similarities or positional biases. As a validation, we compute the Spearman rank correlation between attention weights (averaged across heads) and consistency verification scores on a held-out set; a correlation >0.5 indicates that attention reflects consistency.

## Contribution

(1) CounterConsist, a counterfactual training procedure for multi-image attribute extraction that explicitly forces models to perform cross-image consistency reasoning through artificially introduced conflicts. (2) An auxiliary per-image consistency verification head that provides interpretable conflict scores, enabling post-hoc analysis of model decisions. (3) Empirical demonstration that CounterConsist improves consensus attribute extraction accuracy on IndustryBench-MIPU compared to standard training, particularly for attributes requiring cross-image reasoning.

## Experiment

### Evaluation Setup

| Role | Choice | Rationale (≤12 words) |
|------|--------|------------------------|
| Dataset | IndustryBench-MIPU | Multi-image attribute extraction benchmark (100k products, 4 images per product, 10 attributes). |
| Primary metric | F1-score | Balances precision/recall for attribute extraction. |
| Baseline 1 | Single-image model (CLIP ViT-B/32) | No cross-image reasoning, violates multi-image assumption. |
| Baseline 2 | Average-aggregation multi-image (ResNet-50 backbone, average pooling across images, MLP head) | Simple baseline without counterfactual training. |
| Baseline 3 | GPT-4o (zero-shot with chain-of-thought) | Strong proprietary VLM for multi-image understanding. |
| Ablation 1 | Ours w/o consistency loss | Tests contribution of consistency verification. |
| Ablation 2 | Ours with real-only pool (no synthetic conflicts) | Tests contribution of synthetic conflicts for generalization. |

Training details: 4 NVIDIA V100 GPUs, batch size 32 per GPU, 48 GPU hours total (12 hours wall time). Memory: ~11 GB per GPU.

### Why this setup validates the claim

The setup tests the central claim that counterfactual training with consistency verification improves multi-image attribute extraction by forcing cross-image reasoning. Using IndustryBench-MIPU ensures realistic product images with multiple attributes. The single-image baseline isolates the necessity of multi-image input; the average-aggregation baseline shows the gain from learned cross-attention; GPT-4o provides a strong upper bound. Ablation 1 removes the consistency loss to isolate its effect. Ablation 2 removes synthetic conflicts to test whether the load-bearing assumption (generalization of consistency head) holds with only real conflicts. F1-score is chosen because it captures both exact matches and partial correctness, which is important for attribute values that may have variants. Additionally, we perform a distributional analysis of natural vs. synthetic conflicts in the validation set to verify that the synthetic pool covers realistic conflict patterns. If our method outperforms baselines, especially on cases with conflicting images, the claim is supported. If not, the falsification conditions apply.

### Expected outcome and causal chain

**vs. Single-image model (CLIP)** — On a product where multiple images show conflicting attribute values (e.g., one shows red color, another blue), CLIP processes each image independently and cannot reconcile conflicts, leading to inconsistent or averaged predictions. Our method uses cross-attention to identify the consensus from the majority of consistent images, so we expect a large gap in F1-score on examples with within-product image variability (expected +15% F1 on conflict subsets), but parity on single-image test cases.

**vs. Average-aggregation multi-image** — On a product where one image contains a distracting object that shares features with the target attribute (e.g., a red marker in a background of a blue product), average pooling dilutes the true signal. Our method’s consistency verification can detect the outlier image and down-weight it via attention, so we expect a noticeable gap on examples where one image is partially inconsistent (expected +8% F1), but similar performance on fully consistent sets.

**vs. GPT-4o** — On a product with subtle attribute differences (e.g., shade of color) across images, GPT-4o may rely on its broad training but lack specialized counterfactual training. Our method is explicitly trained to ignore conflicting images, so we expect competitive F1-score on typical examples (within 2% of GPT-4o) but a larger advantage on hard cases with intentional ambiguity (expected +5% F1).

**vs. Ours with real-only pool** — On products where natural conflicts are well-represented in the real pool, performance should be similar. However, on rare or unseen conflict types, the synthetic pool provides additional coverage; we expect our full method to outperform the real-only variant by 3% F1 on the tail of the conflict distribution, confirming the value of synthetic augmentation.

Additionally, we validate the attention-consistency link: we compute Spearman rank correlation between attention weights and consistency verification scores on a held-out set of 500 examples. We expect a correlation >0.5, indicating that attention reflects consistency.

### What would falsify this idea

If our method’s improvement over the average-aggregation baseline is uniform across all subsets rather than concentrated on examples with inconsistent images, that would indicate the consistency verification is not driving the gain. Additionally, if the ablation without consistency loss matches the full method, the centrality of the counterfactual training claim is falsified. If the correlation between attention and consistency is below 0.3, the cross-attention operationalization of cross-image reasoning is not validated. If the real-only pool ablation matches the full method, the synthetic augmentation is unnecessary, weakening the load-bearing assumption.
